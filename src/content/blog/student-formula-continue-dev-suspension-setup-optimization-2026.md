## この記事を読む前に

本ブログの「[Continue.devとMATLAB MCPで始めるオープンソースAIコーディング](/blog/continue-dev-matlab-mcp-open-source-ai-coding-mbd-2026)」でContinue.devのセットアップ方法を紹介しました。この記事ではそれを学生フォーミュラのサスペンション開発に応用し、スプリングレートとスタビライザー剛性の最適化スクリプト生成を自動化します。

## 学生フォーミュラにおける課題

学生フォーミュラチームのサスペンション担当が直面する典型的な状況がある。

「フロントとリアのスプリングレートを変えてロールグラジェントを下げたいが、fminconの使い方がわからず調べながら書いていたら3時間経ってしまった。しかも制約の書き方を間違えて誤った最適解を出してしまった。」

数字で見ると問題の深刻さがわかる：

- fmincon初心者がアンダーステア係数の制約付き最適化スクリプトを書く：**平均3〜4時間**
- 制約式ミスによる誤最適解が発覚して書き直し：さらに**1〜2時間**
- GitHub Copilot全メンバー分の月額：チーム6人で**月1万1,400円**（予算内に収まらない）

予算が限られた学生チームにとって、AIコーディングツールへの継続投資は難しい。Continue.devはこの問題を解決する——本体は完全無料で、AI推論コストはAnthropicの無料枠か、ローカルのOllamaモデルで賄える。

## Continue.devを使った解決アプローチ

Continue.devはVS CodeのサイドパネルからMATLABコードに対してAIに質問・生成を依頼できる。通常のChatGPTにコードを貼り付ける方法と何が違うのか。

**RAG（Retrieval-Augmented Generation：検索拡張生成）**によって、Continue.devはリポジトリ全体をインデックス化し、プロジェクト固有の変数名・関数シグネチャ・Simulinkモデルのインターフェースを理解した上でコードを生成する。

さらにMATLAB MCPサーバーと接続することで、AIがMATLABのワークスペースを直接読み書きできる。「今のモデルにfminconを使った最適化を追加して」という自然言語の指示が、そのまま実行可能なMATLABコードに変換される。

MATLABの最適化ツールボックスには`fmincon`（制約付き非線形最適化）という強力な関数があるが、目的関数と制約関数を別々に定義する構造が初心者には複雑に見える。Continue.devは「この仕様を満たすfminconスクリプトを書いて」と伝えるだけで、正しい構造のボイラープレートを出力する。

## 実装：ステップバイステップ

### 前提条件

- VS Code 1.90以降（無料）
- Continue.dev拡張機能（`code --install-extension continue.continue` でインストール）
- MATLAB R2024a以降（最適化ツールボックス付属）
- AnthropicのAPIキー（月$5相当の無料枠あり）

**最小config.yaml（3分で完了）:**

```yaml
# ~/.continue/config.yaml
# === モデル設定（AnthropicのAPIキーのみ必要） ===
models:
  - name: claude-sonnet-4
    provider: anthropic
    model: claude-sonnet-4-6
    apiKey: ${ANTHROPIC_API_KEY}  # 環境変数から読む（ベタ書き禁止）
    roles: [chat, edit]
```

### ステップ1：車両パラメータをプロンプトで渡す

VS Codeで任意のMATLABファイルを開き、`Ctrl+L` でContinue.devのチャットを起動して以下を送信する：

```
以下の仕様でフォーミュラカーのサスペンション最適化スクリプトをMATLABで書いてください。

目的：コーナリング時のロールグラジェントを最小化する
制約：アンダーステア係数 K_us を -0.005〜0.005 rad/g に収める

設計変数：
  - フロントスプリングレート kf: 15,000〜35,000 N/m
  - リアスプリングレート kr: 20,000〜40,000 N/m
  - フロントスタビライザー剛性 ksf: 0〜5,000 N/m/deg

車両パラメータ：
  - 総重量 m = 280 kg
  - フロント重量配分 wf = 0.45
  - フロントトレッド tf = 1,200 mm
  - リアトレッド tr = 1,150 mm
  - 重心高さ h = 300 mm

fmincon を使い、収束履歴もプロットしてください。
```

### ステップ2：生成されたコードを確認して実行

```matlab
% === ステップ1: 車両パラメータ定義 ===
% 実車に合わせて数値を変更する（ここだけ修正すればよい）
vehicle.m   = 280;       % 車体総重量 [kg]（ドライバー込み）
vehicle.wf  = 0.45;      % フロント重量配分 [-]（0.45 = 前45%, 後55%）
vehicle.tf  = 1.200;     % フロントトレッド [m]
vehicle.tr  = 1.150;     % リアトレッド [m]
vehicle.h   = 0.30;      % 重心高さ [m]（低いほどロールしにくい）

% === ステップ2: 設計変数の初期値と上下限 ===
% x = [kf, kr, ksf]
x0 = [25000, 30000, 2500];   % 初期推定値（経験則から設定）
lb = [15000, 20000,    0];   % 下限（ばね破壊荷重などで決まる）
ub = [35000, 40000, 5000];   % 上限（サスペンションストローク制限）

% === ステップ3: fmincon で最適化を実行 ===
% fmincon: 制約付き非線形最適化アルゴリズム
% 「目的関数を最小化しながら制約式を満たす x を探す」
options = optimoptions('fmincon', ...
    'Display',       'iter', ...      % 各反復の状況を表示
    'MaxIterations', 200, ...
    'OutputFcn',     @convergenceLogger);  % 収束履歴を記録

[x_opt, fval, exitflag] = fmincon(...
    @(x) rollGradient(x, vehicle), ...          % 目的関数
    x0, [], [], [], [], lb, ub, ...
    @(x) understeerConstraint(x, vehicle), ...  % 制約関数
    options);

% === ステップ4: 結果を表示 ===
fprintf('=== 最適化結果 ===\n');
fprintf('フロントスプリングレート: %.0f N/m\n', x_opt(1));
fprintf('リアスプリングレート    : %.0f N/m\n', x_opt(2));
fprintf('フロントスタビライザー  : %.0f N/m/deg\n', x_opt(3));
fprintf('最適ロールグラジェント  : %.2f deg/g\n', fval);
fprintf('終了フラグ              : %d（1=正常収束）\n', exitflag);

% ==== ローカル関数（fminconが内部で呼び出す） ====

function J = rollGradient(x, v)
    % ロールグラジェント [deg/g] を計算する（目的関数）
    % ロール剛性 = スプリング剛性 × (トレッド/2)²
    Kf = (x(1) + x(3)) * (v.tf / 2)^2;  % フロントロール剛性 [Nm/rad]
    Kr =  x(2)         * (v.tr / 2)^2;  % リアロール剛性 [Nm/rad]
    K_total = Kf + Kr;
    roll_1g = (v.m * 9.81 * v.h) / K_total;  % 横1g時のロール角 [rad]
    J = rad2deg(roll_1g);                      % 単位変換 → [deg/g]
end

function [c, ceq] = understeerConstraint(x, v)
    % アンダーステア係数の制約（-0.005 ≤ K_us ≤ 0.005 rad/g）
    % K_us > 0 でアンダー傾向、< 0 でオーバー傾向
    Kf = (x(1) + x(3)) * (v.tf / 2)^2;
    Kr =  x(2)         * (v.tr / 2)^2;
    K_us = v.m * 9.81 * (v.wf / Kf - (1 - v.wf) / Kr);
    c   = [K_us - 0.005; -K_us - 0.005];  % 不等式制約（c ≤ 0 でOK）
    ceq = [];
end

function stop = convergenceLogger(~, optimValues, state)
    % 収束履歴をリアルタイムでプロットする
    persistent fig_handle
    if strcmp(state, 'init')
        fig_handle = figure('Name', '収束履歴');
        xlabel('反復回数'); ylabel('ロールグラジェント [deg/g]');
        title('fmincon 収束履歴'); hold on; grid on;
    end
    if ~isempty(optimValues.fval)
        plot(optimValues.iteration, optimValues.fval, 'b.', 'MarkerSize', 8);
        drawnow;
    end
    stop = false;
end
```

## よくあるエラーと対処

| エラー | 原因 | 解決法 |
|--------|------|--------|
| `Optimization Toolbox is not available` | ツールボックス未ライセンス | 大学のMATLABライセンス管理者に確認する |
| `exitflag: 0`（最大反復数超過） | 問題が複雑すぎる | `MaxIterations` を 500 に増やす |
| `Constraint violation at initial point` | 初期値が制約を満たさない | x0 をロールグラジェント制約内の値に変更する |
| Continue.dev が MATLAB コードを誤生成 | プロンプトが曖昧 | 「MATLAB R2024a の fmincon を使って」と明示する |

## 学生フォーミュラ・レース車両開発への応用

今回のスプリングレート最適化は、学生フォーミュラの典型的なセットアップ課題の一つだ。Continue.devと MATLAB を組み合わせると、さらに次のシナリオに展開できる。

**具体的なシナリオ：テスト走行後24時間で次イベントのセットアップを決定する**

1. 走行会で加速度センサーと角速度センサーのログを取得（100 Hz、走行1セッション約15分分）
2. Continue.devに「このデータからロールグラジェントの実測値を抽出して上の最適化コードに組み込んで」と指示
3. 実測値ベースの最適解が30分以内に出力される
4. 次走行に向けて計算根拠付きでスプリングレートを変更

**背景理論（ロールグラジェントと接地荷重の関係）：**
ロールグラジェント（横加速度 1 g あたりのロール角）が大きいと、外輪への荷重移動が増える。タイヤのグリップはロードセンシティビティ（荷重に対するグリップの感度）により非線形に変化するため、ロールが大きいほど旋回中の有効グリップが下がる。`fmincon` の制約付き最適化はこうした「一方を良くすると他方が悪化する」トレードオフを数値的に解いてくれる。

**実際に動くコードの拡張（実測データを使う場合）：**
目的関数 `rollGradient` の中で、シミュレーション値と計算値の二乗誤差を返すように変更するだけで、データドリブン最適化になる。Continue.devに「目的関数を実測ロールログとの残差最小化に変えて」と指示すれば、10分で拡張版が生成される。

**Before / After（テスト走行データドリブン版）：**

| 指標 | 走行後の従来セットアップ変更 | Continue.dev＋実測データ最適化 |
|------|---------------------------|---------------------------------|
| 変更根拠 | ドライバーのフィーリング頼り | **計算による最適値 ± 5% 以内** |
| 次走行までの準備時間 | 経験者判断で 2 時間 | **実測データ込みで 30 分** |
| 試せるセットアップ候補数 | 1〜2 案 | **10 案以上を数値比較** |

1. Continue.devをインストールして上の完全コードをそのまま実行する（デフォルトパラメータで動く）
2. `vehicle.m = 280` と `vehicle.wf = 0.45` を自チームの実測値に書き換える
