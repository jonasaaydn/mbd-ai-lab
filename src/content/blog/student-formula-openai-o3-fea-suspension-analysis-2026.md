## 学生フォーミュラにおける課題

FEA（Finite Element Analysis：有限要素解析）は学生フォーミュラチームに広く普及していますが、「解析は走らせられるが結果の解釈ができない」という問題が頻発します。

具体的なシナリオ：
- サスペンションロッカーアームのFEA解析で、最大主応力が**降伏応力の112%**という結果が出た。どこを補強すべきか分からない
- 応力集中箇所が3箇所あるが、**どの順番で対策すべきか**優先順位をつけられない
- FEAツール（ANSYS, Abaqus, SimScale等）が出すソルバーレポートを読んでも、「安全率が低い」こと以上の洞察が得られない
- 構造解析の専門家（指導教員）はなかなかつかまらず、**問い合わせから回答まで3〜5日**かかる

結果として「とりあえず板厚を1mm増やす」という非効率な試行錯誤を繰り返し、重量増加とコストアップにつながります。

o3が持つ**理工学的な推論深度**はこの課題に直接対応します。応力集中の物理的メカニズム、材料力学の背景理論、製造制約を考慮した現実的な対策案を、テキストとして渡したFEA結果から生成できます。

## OpenAI o3を使った解決アプローチ

**背景理論：なぜo3が構造解析結果の解釈に有効か**

FEA結果の解釈には複数の知識領域を横断する推論が必要です：

1. **材料力学**：応力テンソル（物体内部の各方向への力の状態）、主応力（最大・最小の法線応力）、Von Mises応力（降伏判定基準）の関係
2. **破壊力学**：応力集中係数（Kt）、切り欠き効果、溶接熱影響域（HAZ：Heat Affected Zone）の強度低下
3. **設計論**：剛性と軽量化のトレードオフ、製造可能性（溶接アクセス性、曲げ半径など）
4. **レース車両特有の要件**：動的荷重（走行時の衝撃係数）、重量規定、コスト制約

o3は**思考の連鎖（Chain-of-Thought Reasoning）**と呼ばれる推論方式により、これらを順序立てて組み合わせた分析ができます。GPT-4oと比べて特に「なぜその部位が問題か」という因果推論が格段に精度高くなります。

## 実装：ステップバイステップ

**前提条件：**
- OpenAI APIキー（o3-miniまたはo3モデルへのアクセス）
- FEAツール（SimScale / ANSYS Workbench / Abaqus等）の実行済み解析結果
- Python 3.10以上・`openai` パッケージ（`pip install openai`）

### ステップ1：FEA結果をo3に渡せる形式に整理

```python
# === ステップ1: FEA結果サマリーを構造化テキストとして準備 ===
# o3はテキスト入力を受け付けるため、FEA結果をCSVまたは構造化テキストにまとめる

def build_fea_summary(component_name: str, material: str,
                      yield_stress_mpa: float, load_cases: list,
                      stress_hotspots: list, displacement_max_mm: float) -> str:
    """FEA結果を o3 が読みやすい構造化テキストに変換する"""
    summary = f"""
=== FEA解析結果レポート ===
部品名: {component_name}
材料: {material}
降伏応力: {yield_stress_mpa} MPa

--- 荷重条件 ---
"""
    for lc in load_cases:
        summary += f"  {lc['name']}: {lc['description']}\n"

    summary += "\n--- 応力ホットスポット（Von Mises応力上位） ---\n"
    for i, spot in enumerate(stress_hotspots):
        safety = yield_stress_mpa / spot["stress_mpa"]
        flag   = " ⚠️ 降伏超過!" if safety < 1.0 else (" ⚡ 要注意" if safety < 1.3 else "")
        summary += (f"  #{i+1} Node {spot['node_id']}: "
                    f"{spot['stress_mpa']:.0f} MPa  "
                    f"安全率={safety:.2f}{flag}  "
                    f"位置={spot['location']}\n")

    summary += f"\n最大変位: {displacement_max_mm:.2f} mm\n"
    return summary


# 実際のFEA解析結果を入力（SimScale/ANSYSのGUIから数値をコピー）
fea_text = build_fea_summary(
    component_name   = "フロントサスペンション ロッカーアーム（6061-T6アルミ）",
    material         = "A6061-T6 (降伏応力 276 MPa, 引張強さ 310 MPa)",
    yield_stress_mpa = 276.0,
    load_cases       = [
        {"name": "LC1 コーナリング最大荷重",
         "description": "横力 2.1kN + 縦力 0.8kN + 垂直力 1.2kN（バンプ時）"},
        {"name": "LC2 ブレーキング",
         "description": "縦力 2.8kN + 垂直力 1.5kN"},
    ],
    stress_hotspots  = [
        {"node_id": 1247, "stress_mpa": 312,
         "location": "ブラケット穴エッジ（R=0.5mm）"},
        {"node_id": 1831, "stress_mpa": 289,
         "location": "板厚変化部（3mm→5mm 段差）"},
        {"node_id": 902,  "stress_mpa": 198,
         "location": "プッシュロッド結合ボルト穴"},
    ],
    displacement_max_mm = 4.8  # 許容値: 2.0mm 超過
)

print(fea_text)
```

### ステップ2：o3に根本原因分析と設計改善案を依頼

```python
# === ステップ2: OpenAI o3 API で根本原因分析と設計改善案を取得 ===

from openai import OpenAI

client = OpenAI(api_key="your-openai-api-key")  # 環境変数推奨: os.environ["OPENAI_API_KEY"]

SYSTEM_PROMPT = """あなたはレース車両の構造設計・FEA解析の専門家です。
学生フォーミュラチーム（大学チーム、予算50万円以下、製造はTIG溶接と機械加工）向けに、
FEA結果の根本原因分析と具体的な設計改善案を提案してください。
制約：2週間以内に実装可能、重量増加を最小限に、製造コスト追加なし。"""

USER_PROMPT = f"""以下のFEA解析結果について、3つの質問に答えてください。

{fea_text}

【質問1】各ホットスポットの応力集中が発生している物理的な根本原因を、
材料力学の観点から説明してください（学部3年生向けの言葉で）。

【質問2】対策の優先順位（高→低）をつけ、それぞれ「なぜその優先順位か」を理由とともに説明してください。

【質問3】最優先ホットスポットに対して、TIG溶接と汎用機械加工で実現できる具体的な設計変更案を
3つ提案してください。各案について：重量変化（+/-g）、製作工数（時間）、期待効果（安全率の改善値）を明記してください。"""

# o3での推論実行
response = client.chat.completions.create(
    model="o3",
    messages=[
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user",   "content": USER_PROMPT},
    ],
)

analysis = response.choices[0].message.content
print("=== o3 FEA解析レポート ===")
print(analysis)

# 結果をMarkdownファイルに保存
with open("fea_improvement_report.md", "w", encoding="utf-8") as f:
    f.write(f"# FEA改善提案レポート\n\n## 入力データ\n```\n{fea_text}\n```\n\n## o3解析結果\n\n{analysis}")
print("\n[完了] fea_improvement_report.md に保存しました")
```

**o3の出力例（抜粋）：**
```
【質問1 根本原因分析】

ホットスポット#1（ブラケット穴エッジ、R=0.5mm）：
R=0.5mmという極小のフィレット半径では、理論応力集中係数Kt≈3.0〜3.5
となり、公称応力の3倍以上の局所応力が発生します。板厚が薄い（3mm）
部位では荷重経路が穴端部に集中するため、特に顕著です。

ホットスポット#2（板厚変化部の段差）：
3mmから5mmへの急激な板厚変化は曲げ剛性の不連続を生みます。
滑らかなテーパー移行がない場合、応力集中係数はKt≈1.8〜2.2に達します。

【質問2 対策優先順位】

優先度1: ホットスポット#1（安全率0.88）
→ 降伏を既に超過。走行中に塑性変形が進行中の可能性があります。
  今週末のテストまでに対策必須。

【質問3 設計変更案（ホットスポット#1）】

案A：フィレット半径の拡大（R0.5mm→R3.0mm）
  重量変化: +3g  製作工数: 0.5時間  安全率: 0.88→1.76

案B：補強ガセット追加（2mm厚アルミ板をTIG溶接）
  重量変化: +18g  製作工数: 2時間  安全率: 0.88→1.45（HAZ考慮後1.16）

案C：肉抜き形状最適化（穴を楕円形に変更）
  重量変化: -5g  製作工数: 1.5時間  安全率: 0.88→1.35
```

### ステップ3：改善案の定量比較テーブルを自動生成

```python
# === ステップ3: 改善案の比較テーブルを自動生成 ===
# o3の出力を構造化して意思決定を支援

COMPARISON_PROMPT = """
先ほど提案した3つの設計変更案について、
以下の評価軸でスコアリングしてください（各5点満点）。
その後、総合推奨案を1つ選んで理由を説明してください。

評価軸:
1. 構造効果（安全率の改善度）
2. 実装速度（2週間以内に完了できるか）
3. 重量影響（軽量化方向が高スコア）
4. リスク（製造失敗・新たな問題発生の可能性が低いほど高スコア）
5. コスト（追加材料費・外注費なし＝5点）

表形式（Markdown）で出力してください。
"""

response2 = client.chat.completions.create(
    model="o3",
    messages=[
        {"role": "system",    "content": SYSTEM_PROMPT},
        {"role": "user",      "content": USER_PROMPT},
        {"role": "assistant", "content": analysis},
        {"role": "user",      "content": COMPARISON_PROMPT},
    ],
)
print(response2.choices[0].message.content)
```

## よくあるエラーと対処

| エラー・問題 | 原因 | 対処方法 |
|------------|------|---------|
| o3の回答が抽象的で具体性に欠ける | プロンプトに製造制約が不足 | チームの設備（TIG溶接機の有無、CNC有無）と予算上限を明示する |
| `RateLimitError` | APIレートリミット超過 | `time.sleep(60)` を挿入、またはo3-miniに切り替えてコスト削減 |
| 提案された改善案がFEAツールで再現できない | 3D形状の説明が不足 | FEA結果のスクリーンショットをテキスト説明に添付（o3はマルチモーダル対応） |
| 同じFEA結果で毎回異なる提案が返る | o3の確率的な推論 | `seed` パラメータを固定、または3回実行して多数決 |
| APIコストが高い | o3の推論トークン消費が大きい | まずo3-miniで概要を取得し、最終確認のみo3を使う |
