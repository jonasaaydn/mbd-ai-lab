## あなたのサロゲートモデルは「使い捨て」になっていないか

レース車両のCAEエンジニアなら、こんな経験があるはずだ。

- フロントウィング形状の空力サロゲートモデルを半年かけて構築した
- 翌シーズン、コンセプトが一新されてモデルが丸ごとお蔵入りに
- また0から2,000件のCFDを回して再訓練

現在の典型的なサロゲートモデル開発は **「1プロジェクト＝1モデル」の使い捨て構造** だ。これはGPTが登場する前の自然言語処理が「1タスク＝1モデル」だったときと同じ構造的限界を抱えている。

英国のスタートアップ・PhysicsXは「これをLLM化できないか」という問いに正面から挑んでいる。

## Large Physics Model（LPM）とは何か

PhysicsXのコンセプトはシンプルだ：

> ChatGPTが大量のテキストで「汎用的な言語理解」を獲得したように、大量の物理シミュレーションデータから「汎用的な物理理解」を学ばせる——それがLarge Physics Model（LPM）だ。

LLMがゼロショットで未知の文書を要約できるように、LPMは **学習時に見ていない形状・境界条件** でも高精度で物理挙動を予測することを目指す。

従来サロゲートモデルは「ある特定の形状パラメータ範囲内でしか使えない」という根本的な制約があった。その制約を、スケールで突き破ろうとしている。

## 従来アプローチとLPMの比較

| 項目 | 従来サロゲートモデル | Large Physics Model |
|------|-------------------|-------------------|
| 学習データ規模 | 数百〜2,000件／プロジェクト | 20,000件超（複数形状横断） |
| 適用範囲 | 1形状コンセプト限定 | 設計空間を横断 |
| 形状変更時の対応 | ほぼゼロから再訓練 | ファインチューニングのみ |
| 分布外予測精度 | 低い（外挿が弱い） | 高い汎化を目標 |
| 開発コスト（初期） | 数ヶ月〜半年 | 数日〜数週間（ファインチューニング） |

## PXNetCar：2万件CFDの「大規模コーパス」

PhysicsXが2026年3月に発表したPXNetCarは、LPM構築のための研究基盤データセットだ。

- **250以上**の異なるベースライン車両デザイン
- **2万件超**のCFDシミュレーション（外部空力）
- 既存データセット（DrivAerNet++、Luminary SHIFT-SUVなど）が抱える「分布外評価の弱さ」を指摘し、自らその克服を目指す設計

この規模感は、従来のプロジェクト単位開発とは次元が違う。

## 実用展開済み：Siemensとのブレーキ冷却最適化

現時点で最も具体的な商用事例は、SiemensとのパートナーシップによるブレーキダクトのCFD最適化ワークフローだ。

**従来のワークフロー：**

```
100〜200件のダクト形状バリエーション生成
 → CFDメッシュ生成（各1〜2時間）
 → 流体解析実行（各4〜8時間）
 → 結果評価・選定
合計：数百〜数千時間のCFD計算
```

**PhysicsX LPMを用いたワークフロー（概念）：**

```python
import physicsx  # 商用API（要契約）

# 事前学習済みモデルをロード
model = physicsx.load_model("automotive-thermal-v1")

# ブレーキダクトのSTLバリエーションをバッチ入力
geometries = load_stl_variants("brake_duct_variants/*.stl")

# 熱流体場を推論（数分で完了）
results = model.predict_thermal_flow(
    geometries=geometries,
    boundary_conditions={
        "inlet_velocity": 30.0,    # m/s
        "ambient_temp": 298.15,    # K
        "brake_heat_flux": 50000,  # W/m²
    }
)

# 最高温度が最小のダクト形状を選定
best = results.rank_by("peak_brake_temp")[0]
print(f"推定ブレーキ最高温度: {best.peak_temp:.1f} K")
```

数週間のCFDキャンペーンを数分の推論に圧縮するのが目標だ。

## 注意点：現時点でできないこと

- **一般公開APIはなし**：現在はSiemens・自動車OEMとの直接商用契約ベースのみ
- **精度の独立検証は限定的**：PXNetCar論文は2026年3月発表でピアレビュー途中
- **対応物理領域は限定的**：現在は主に外部空力・熱流体。構造FEM・車両ダイナミクスは未対応
- **最終確認はフルCFDが依然必須**：高精度を要求される最終設計確認フェーズは代替不可
