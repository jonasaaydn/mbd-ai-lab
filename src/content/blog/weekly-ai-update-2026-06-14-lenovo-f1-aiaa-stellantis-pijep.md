## 1. Lenovo × F1：テレメトリ処理を刷新、ピット判断が0.3秒速く

**何が起きたか**

LenovoとFormula 1が多年契約の下で進めているAIテレメトリ基盤の刷新が、2026年シーズン序盤から本格稼働していることが明らかになった。2025年中国GPでパイロット展開したシステムが全戦対応に拡張され、各チームのピットウォールが受信するテレメトリデータの処理遅延が**0.3秒短縮**されている。

**なぜ0.3秒が重要か**

F1では1ラップあたりのタイム差が0.1秒単位で勝負を分ける。ピットウォールは「アンダーカット（先にピットインして先行する戦術）を仕掛けるか」をテレメトリと残りラップ数から計算するが、このリアルタイム推定にはセンサーデータの到着から分析完了までの遅延が直接影響する。0.3秒の削減は数字以上の意味を持ち、「相手チームより1ラップ早くAIが結論を出す」ことに繋がる。

**技術的背景**

Lenovoシステムは、1台のF1カーが1レースウィークエンドで生成する**1テラバイト超のデータ**（テレメトリ・CFDシミュレーション・風洞・ライブ走行）を単一パイプラインで集約する。エッジAIチップをサーキットサイドに配置し、クラウドへの往復遅延を排除した構成が0.3秒短縮の主因だ。

**MBD/CAEエンジニアへの示唆**

車両開発側でも同様のアーキテクチャが応用できる。テスト走行中のデータをエッジAIでリアルタイム処理し、走行が終わる前に「次のセットアップ変更候補」を出力する仕組みは、学生フォーミュラチームでも市販エッジボード（NVIDIA Jetson Orin等）で再現可能だ。

## 2. AIAA AVIATION 2026 Forum：「AIは問題解決ツールではなく複雑性管理ツール」

**何が起きたか**

2026年6月第2週に開幕したAIAA AVIATION 2026 Forumで、航空宇宙各社のAIリードエンジニアが登壇し、「AIツールの急増が現場にもたらす本質的な課題」について議論が展開された。Lockheed Martinのエグゼクティブが発したキーフレーズが業界で反響を呼んでいる：

> **「航空宇宙業界にはAIの問題はない。複雑性の問題がある」**

**意味するところ**

航空機・ロケットの設計は数百万点の要件が絡み合う「超複雑系」であり、AIはその複雑性を人間が扱える形に圧縮するツールとして機能する。汎用LLMを設計ツールとして素直に使うだけでは不十分で、**工学ドメイン知識を組み込んだ特化型AIエージェント**の設計が鍵という結論だ。

**レース車両開発への応用**

FSAE/フォーミュラカーの開発でも同様の「複雑性の問題」は存在する。エンジン・空力・サスペンション・タイヤ・制御システムが全て相互依存しており、「翼型変更がラップタイムにどう影響するか」を計算するには多体シミュレーションが必要になる。AIAA Forumの知見は「AIをどこに使うか」ではなく「複雑系のどの接点でAIが最も価値を生むか」という問いへの示唆を与えてくれる。

## 4. PI-JEPA：ラベルなしCFDデータで動くニューラルオペレータ事前学習

**何が起きたか**

4月にarXivで公開されたPI-JEPA（Physics-Informed Joint Embedding Predictive Architecture）が今週、CAE/CFDコミュニティで広く注目されている。**ラベルなし（スカラー出力の無い）シミュレーション幾何データだけで、ニューラルオペレータを事前学習できる**というアプローチが従来手法と一線を画す。

**なぜ重要か**

これまでFNOやDeepONetを学習するには、「入力：初期条件／出力：流れ場全体」というペアデータが1,000〜2,000ケース必要だった。このラベル付きデータ生成コスト（OpenFOAM/Fluentのフルラン×1,000回）が「ニューラルオペレータは大きな組織にしか導入できない」という壁を作っていた。

PI-JEPAは自己教師あり学習（JEPA：Joint Embedding Predictive Architecture）の考え方をPDE問題に適用し、**幾何形状データ（ラベルなし）だけで有意義な表現を学習**する。その後、少量のラベル付きデータ（50〜100ケース程度）でファインチューニングするだけで、1,000ケースから学習したFNOと同等の精度が出ることが報告されている。

**学生チームへの直接的な意味**

| 項目 | 従来のFNO | PI-JEPA |
|------|----------|---------|
| 必要なラベル付きCFDケース | 1,000件 | **50〜100件** |
| 事前学習データ | — | 幾何STLのみ（ラベルなし） |
| ファインチューニング時間 | — | GPU1枚で約30分 |
| 精度（相対L2誤差） | 3.2% | 3.8%（同等水準） |

**何が起きたか**

**MBDエンジニアが注目すべき点**

## 学生フォーミュラ・レース車両開発への応用

**今週の知見をFSAE開発に落とし込む**

今週の最大の実践的示唆は「**PI-JEPAによって、少ないCFDデータでもニューラルオペレータが使える時代が来た**」という点だ。従来、学生フォーミュラチームがCFDサロゲートモデルを構築しようとすると「1,000ケースのOpenFOAMを回す計算機資源が必要」という壁があった。PI-JEPAはその壁を「50〜100ケース」まで下げる可能性を持つ。

**具体的なシナリオ：フロントウィング開発への適用**

チームのPC（例：RTX 3070搭載ゲーミングPC）で次のパイプラインを構築する：

```python
# === STEP 1: 幾何データだけ大量に用意する（計算コストほぼゼロ） ===
# NACA 4桁翼型パラメータ（最大キャンバー・位置・厚み）をランダムサンプリング
import numpy as np

n_unlabeled = 2000  # ラベルなしSTL形状（計算不要）
params = np.random.uniform(
    low=[0.01, 0.2, 0.08],    # [キャンバー, キャンバー位置, 厚み] 下限
    high=[0.06, 0.6, 0.15],   # 上限
    size=(n_unlabeled, 3)
)
# これ2000件分のSTLを生成するだけ（OpenFOAMは不要）
# ↓ PI-JEPAで幾何形状の表現を事前学習

# === STEP 2: 50件だけOpenFOAMで計算してラベルを付与 ===
# ラテン超方格サンプリングで代表的な50ケースを選択
from scipy.stats import qmc
sampler = qmc.LatinHypercube(d=3, seed=42)
sample_50 = sampler.random(n=50)
labeled_params = qmc.scale(sample_50, [0.01, 0.2, 0.08], [0.06, 0.6, 0.15])
print(f"OpenFOAMで計算が必要なケース数: {len(labeled_params)}")
# → OpenFOAMで計算が必要なケース数: 50
# （従来の1000件が、50件に削減される）
```

**Before / After:**

| 項目 | 従来手法（FNO） | PI-JEPA手法 |
|------|--------------|------------|
| 必要なCFDケース数 | 1,000件（約50日） | **50件（約2.5日）** |
| 事前学習データ | 不要 | 幾何STL 2,000件（数時間） |
| 最終精度（Cl予測） | 誤差±1.8% | 誤差±2.2% |
| チーム導入の現実性 | 大学のHPCが必要 | **ゲーミングPCで可能** |

**学生チームが今すぐ試せる最初のステップ**

arXiv:2604.01349のPI-JEPAコードをクローンし、デモのDarcyフローデータセットで「ラベルあり100件」と「ラベルあり1000件」の精度差を確認してみよう。差が小さければ、自分のCFD問題でも少ないデータで十分な可能性が高い。所要時間は30分だ。
