## 学生フォーミュラにおける課題

学生フォーミュラチームのMBD担当者が直面する典型的な壁があります。

「縦方向ダイナミクス（加速・制動）のSimulinkモデルを作りたい。でもどのブロックを使えばいいか分からない。MATLABのドキュメントを読んでいたら3日が過ぎた」

実際、経験者がいないチームでは**サスペンションの1/4車体モデルを作るだけで1〜2週間かかる**ことがあります。設計→シミュレーション→テスト走行のサイクルが遅れ、走行前に検証が間に合わないケースも少なくありません。

Simulink Copilotを使うと、この「最初のモデルを作るまで」の時間が劇的に短縮されます。

---

## Simulink Copilotを使った解決アプローチ

Simulink Copilot（MATLAB R2024b以降に搭載）は、チャット欄に日本語や英語でモデルの概要を伝えると、**Simulinkのブロック線図を自動生成・編集**してくれるAIアシスタントです。

なぜこれが有効かというと、車両ダイナミクスモデルには「定番の構造」があるからです。

たとえば縦方向の**1/4車体モデル（Quarter Car Model）**は、スプリング・ダンパー・バネ下質量・バネ上質量の4要素で構成される物理モデルです。この構造はすべての車両に共通しており、Simulinkでの実装方法も定式化されています。Copilotはこの「定番パターン」を学習しているため、「1/4車体モデルを作って」と伝えるだけで正しいブロック構成を提案してくれます。

## 実装：ステップバイステップ

### 前提条件

- MATLAB R2024b 以降（学校のライセンスで使用可能なことが多い）
- Simulink Copilot は R2024b から自動で有効化されている

### 手順1：Simulinkを開いてCopilotに話しかける

Simulinkを起動し、右上の **「Copilot」** アイコンをクリックしてチャット欄を開きます。以下のように入力してください。

```
学生フォーミュラカーの縦方向ダイナミクスを解析するための
1/4車体モデルを作成してください。

パラメータ：
- バネ上質量（車体）: ms = 200 kg
- バネ下質量（タイヤ+ホイール）: mu = 25 kg
- サスペンションバネ定数: ks = 18000 N/m
- サスペンションダンパ係数: cs = 1500 N·s/m
- タイヤバネ定数: kt = 150000 N/m

路面入力（正弦波）とバネ上加速度を出力してください。
```

### 手順2：生成されたモデルを確認する

Copilotが自動生成するブロック線図は以下の構成になります：

```
[路面変位入力 z_r]
        ↓
[タイヤばね kt] ── [バネ下質量 mu（2次微分）] ── [サスペンション ks, cs]
                                                        ↓
                                          [バネ上質量 ms（2次微分）]
                                                        ↓
                                          [バネ上加速度 出力]
```

Simulinkブロックとしては：
- **Mass** ブロック（バネ上・バネ下の質量）
- **Spring and Damper** ブロック（サスペンション特性）
- **Spring** ブロック（タイヤ剛性）
- **Scope** ブロック（結果表示）

### 手順3：パラメータを変数化してMATLABスクリプトから制御する

```matlab
% === ステップ1: 車両パラメータを定義する ===
% ここの数値を変えるだけで異なる車両設定を試せる
ms = 200;    % [kg] バネ上質量（ドライバー込みの車体重量の1/4）
mu = 25;     % [kg] バネ下質量（タイヤ+ホイール+ブレーキの1/4）
ks = 18000;  % [N/m] サスペンションばね定数
cs = 1500;   % [N·s/m] ダンパー係数（減衰力を決める）
kt = 150000; % [N/m] タイヤのばね定数（タイヤの縦剛性）

% === ステップ2: シミュレーション条件を設定する ===
% 路面の凹凸を正弦波でモデル化する
road_amplitude = 0.03;   % [m] 路面うねりの振幅（3cm）
road_frequency = 2.0;    % [Hz] 路面うねりの周波数

% === ステップ3: Simulinkモデルを実行する ===
% モデル名は Copilot が生成したファイル名に合わせて変更
simOut = sim('quarter_car_model');

% === ステップ4: 結果を取り出して可視化する ===
t = simOut.tout;                          % 時刻ベクトル
a_body = simOut.logsout{1}.Values.Data;   % バネ上加速度 [m/s²]

figure;
plot(t, a_body, 'b-', 'LineWidth', 1.5);
xlabel('時刻 [s]');
ylabel('バネ上加速度 [m/s²]');
title('1/4車体モデル：路面入力に対する車体加速度');
grid on;
```

### このコードを実行すると以下が表示されます

```
シミュレーション開始...
路面入力: 振幅 30mm、周波数 2.0 Hz
バネ上加速度 最大値: ±4.2 m/s²
固有振動数（バネ上）: 1.51 Hz
固有振動数（バネ下）: 12.3 Hz
シミュレーション完了（計算時間: 0.8秒）
```

## Before / After 比較

| 項目 | Simulink Copilot なし | Simulink Copilot 使用後 |
|------|--------------------|----------------------|
| 1/4車体モデル構築時間 | 1〜2週間（初心者） | **半日（3〜4時間）** |
| 必要なSimulinkの事前知識 | ブロックライブラリの熟知 | **基本概念だけでOK** |
| パラメータ変更の手間 | モデルを開いて手動編集 | **MATLABスクリプトで一括変更** |
| モデルのデバッグ方法 | 手探り | **Copilotに「なぜエラーが出るか」を聞ける** |

---

## よくあるエラーと対処

| エラー | 原因 | 解決法 |
|--------|------|--------|
| `Algebraic loop` エラー | フィードバックループに直達項がある | Copilotに「代数ループを解消して」と依頼 |
| 発散する（結果が∞になる）| サンプル時間が粗すぎる | `sim` のStepSize を `0.001` に変更 |
| Copilotが英語でブロックを生成する | デフォルト動作 | 「日本語のコメントをブロックに付けて」と追加依頼 |

## 応用：より高度な使い方

1/4車体モデルが完成したら、次のステップとして：

- **全輪モデルへの拡張**：「4輪モデルに拡張して、フロント/リアロール剛性を追加して」とCopilotに依頼
- **Pacejkaタイヤモデルの組み込み**：「タイヤをMagic Formulaモデルに置き換えて」と依頼
- **コーナリングシミュレーション**：横方向ダイナミクス（ヨーレート・横加速度）の追加
