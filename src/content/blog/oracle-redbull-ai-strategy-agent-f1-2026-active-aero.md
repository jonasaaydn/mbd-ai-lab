## はじめに

2026年のF1グランプリでは、エンジニアが紙に書いた計算でピットタイミングを決めていた時代は完全に終わった。アクティブエアロダイナミクス（可変フロント＆リアウイング）、燃焼/電気エネルギー50:50マネジメント、新型E10持続可能燃料——3つの変数が同時にリアルタイムで動き続ける2026年レギュレーションのもとでは、人間の判断スピードがボトルネックになる。

「AIを使う」ではなく「AIなしでは戦えない」状況に初めてなったのが2026年シーズンだ。Red Bull Racingは2026年2月にOracleとの複数年契約を延長し、AI戦略エージェントを今季から本格投入することを公式発表した。毎レース**40億回のモンテカルロシミュレーション**、毎秒**100万超のデータポイント**——これがF1ピットウォールの現実だ。

## F1 2026規則変更とAIシミュレーションの関係

2026年F1技術規則の変更点は、ひとことで言えば「不確実性の次元が増えた」ことを意味する。

**アクティブエアロダイナミクス（Active Aero）**：フロント・リアウイングのフラップ角度がストレート/コーナーで動的に変化する。DRSの概念が拡張され、指定ストレートでは条件なしにリアウイングが開放される。ただしフラップが開く/閉じるタイミングと角度をどう組み合わせるかで、同一ラップでも燃費・タイヤ負荷・タイム目標が変わる。

**ハイブリッドパワー50:50**：内燃エンジンと電気モーターの出力比がほぼ等しくなり、エネルギー回収（ERS）の展開戦略がラップ全体を通じて細かく設計される。ブレーキングポイント、ERS放電タイミング、アクティブエアロ角度——これら3変数が連動するため、最適戦略の探索空間は従来比で数百倍に拡大した。

**結果として**、従来の戦略ツールでは計算が間に合わない。リアルタイムにAIが代替シナリオを生成し続ける体制が不可欠になった。

## Oracle AI戦略エージェントの実際の動作

Oracle × Red Bull Racingで稼働するAI戦略エージェントの主要機能を分解する。

### レースウィーク前：40億シミュレーション

レース週末前にチームは約**40億回のモンテカルロシミュレーション**を実行する。入力変数は以下のとおりだ。

- タイヤコンパウンド選択（C1〜C5）とデグラデーションカーブ
- エネルギー展開ウィンドウ（各コーナー単位）
- アクティブエアロ設定（ストレートあたりの開放タイミング）
- セーフティカー確率（過去レースデータから推定）
- ライバル車の戦略予測（テレメトリ履歴ベース）

練習走行後には最新テレメトリを統合して再シミュレーションを行い、予選・決勝のグリッドポジション別シナリオを更新する。

### レース中：毎秒100万データポイントの処理

2026年の車両は300〜600個のオンボードセンサーを搭載し、空力・機械・電気・熱・ドライバー入力の全チャンネルを合わせて**毎秒100万超のデータポイント**をストリーミング送信する。OracleのAI戦略エージェントはこのデータをリアルタイムに処理し、次のアクションをピットウォールに提示する。

```
[リアルタイム提示例]
セクター2完了時点（Lap 23）：
  ・現在タイヤデグラデーション: 予測より+2.3%速い
  ・エネルギー消費: 計画対比 +4.7% (アクティブエアロ開放頻度が原因)
  ・推奨: Lap 26でピット（0.8秒のアンダーカット窓あり）
  ・代替: Lap 29でピット（上位3台がLap 27前後にピットした場合）
  ・確信度: 72% vs 21% (残り7%: 確認不能シナリオ)
```

### セーフティカー・天候変化への即応

従来、エンジニアが「セーフティカーが出た！」と判断してピットインを指示するまでに10〜15秒のラグがあった。AIエージェントは**0.3秒以内**に「今すぐピット」「このまま走行」「ソフト換装でオーバーカット狙い」の3案を優先順位付きでスクリーンに表示する。

## チーム別AIアーキテクチャの違い

| チーム | AIパートナー | 特徴 |
|--------|-------------|------|
| Red Bull Racing | Oracle AI | モンテカルロ×アジェンティックAI、ピットウォール直結の意思決定支援 |
| Mercedes | G42 + SAP + Meta AI | 予測アルゴリズム＋エンタープライズデータ統合、複数AIレイヤー構成 |
| Ferrari | Amazon SageMaker | カスタムMLモデル、CFD計算を最大60%高速化 |
| McLaren | SAP + 独自開発 | 軽量リアルタイムモデル、ドライバーフィードバックループを重視 |

Oracleのアプローチが他と異なるのは「自動化」ではなく「人間の意思決定を補強する」設計思想だ。AIエージェントが提示するのはあくまで「推奨と根拠」であり、最終判断はStrategyエンジニアが行う。

## Dell製トラックサイドマイクロデータセンターとの連携

2026年からDellはF1の公式テクノロジーパートナーとして、トラックサイドに展開できる**マイクロデータセンター**を各チームに提供している。このエッジ処理基盤により、クラウドへの往復レイテンシなしにリアルタイムデジタルツイン更新が可能になった。

```
[データフロー]
車両センサー
  → トラックサイドDellエッジサーバー（<2ms処理）
    → Oracle AI戦略エージェント（シナリオ更新）
      → ピットウォールスクリーン（0.3秒以内に表示）
```

以前はクラウド往復で200〜500msかかっていたデジタルツイン更新が、エッジ処理で10ms以下に短縮された。これがセーフティカー瞬時対応の技術的根拠だ。

## 注意点・落とし穴

**AIはあくまで「提案」**：OracleのAI戦略エージェントはピットウォールに「推奨」を表示するが、実行の決定権はエンジニアにある。2026年モナコGPでは、AIが「ピット推奨」を出しながらチームが戦略を変えてポジションを守ったケースがあった。「AIに従えば正解」ではなく「AIの根拠を読んで判断する」スキルがエンジニアに求められる。

**データ品質がすべてを左右する**：40億シミュレーションの精度は入力データの質に依存する。センサーノイズ・温度ドリフト・CAN通信エラーがあると、AIが出す確信度スコアが実態より高く出る危険性がある。データパイプラインの品質管理がAI投資の前提条件だ。

**ブラックボックスリスク**：戦略エンジニアがAIの推奨理由を理解できない場合、「なぜそこでピットしたのか」のレビューが困難になる。Explainable AI（XAI）の組み込みと、エンジニアがシナリオを手動で展開できるオーバーライド機能の整備が各チームの課題になっている。

## 応用：レース開発エンジニアとしての学び方

F1のAI戦略エージェントの構造は、レーシングカー開発のシミュレーション業務にも転用できる。

- **モンテカルロ法**によるサスペンションセットアップ探索（不確実パラメータを確率分布で表現）
- **デジタルツイン**とセンサーデータの組み合わせで、車両状態をリアルタイム推定
- **エッジAI**によるHILテスト中のリアルタイム異常検知

Pythonで始めるなら `scipy.stats` のモンテカルロと `ray` による並列化が最初の一歩になる。

## 今すぐ試せる最初の一歩

モンテカルロ戦略シミュレーションの最小実装を試す。

```python
import numpy as np

# レース戦略の簡易モンテカルロシミュレーション（1000回）
np.random.seed(42)
N = 1000
lap_total = 57

# タイヤデグラデーション（ラップタイム劣化率）を確率分布で表現
deg_mean, deg_std = 0.08, 0.02  # 秒/ラップ

best_lap_for_pit = []
for _ in range(N):
    deg = np.random.normal(deg_mean, deg_std)
    # ピットタイミングを変えてラップタイム合計を最小化
    min_total = np.inf
    best_pit = 20
    for pit_lap in range(15, 40):
        stint1 = sum(30.0 + deg * i for i in range(pit_lap))
        stint2 = sum(30.0 + deg * i for i in range(lap_total - pit_lap))
        pit_loss = 22.0  # ピットロスタイム（秒）
        total = stint1 + stint2 + pit_loss
        if total < min_total:
            min_total = total
            best_pit = pit_lap
    best_lap_for_pit.append(best_pit)

print(f"最適ピットラップ: 平均 {np.mean(best_lap_for_pit):.1f} ラップ目")
print(f"  ±1σ範囲: {np.mean(best_lap_for_pit)-np.std(best_lap_for_pit):.1f} 〜 "
      f"{np.mean(best_lap_for_pit)+np.std(best_lap_for_pit):.1f}")
```

このコードを実行するだけで5分後には「なぜF1チームがモンテカルロシミュレーションを使うのか」が体感できる。
