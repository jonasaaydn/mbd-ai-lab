## 学生フォーミュラにおける課題

学生フォーミュラのサスペンション設計では、以下の問題が頻繁に発生します。

- **ジオメトリパラメータの組み合わせ爆発**: キャンバー・トー・キャスター・KPI（キングピンインクリネーション：タイヤ転舵軸の傾き角）を個別に変えると試したい組み合わせは50通り以上
- **スクリプト記述の壁**: 設計担当の2〜3年生はMATLABに不慣れで、手動でスクリプトを書くのに1回あたり30〜60分かかる
- **テスト前日の時間不足**: ドライバーと設計チームで設定を議論したいが、解析に時間がかかりすぎて結論が出ない

ある学生フォーミュラチームの実態：「サスペンションジオメトリ確認1パラメータ変更あたり20分 × 20設定 = 延べ6〜7時間」という状況が、テスト前日に毎回発生しています。Clineを使えば、自然言語の指示だけでMATLABコードの生成・実行・結果の可視化まで自動化できます。

## ClineとMATLAB MCPを使った解決アプローチ

Clineは VS Code 上で動作するオープンソースのAIコーディングエージェントで、**無料で使用できます**（APIトークンコストのみ発生、またはOllamaでのローカルモデル使用で完全無料）。

MATLAB MCP（Model Context Protocol）サーバーと組み合わせると、ClineはMATLABのワークスペースに直接アクセスし、スクリプトの生成・実行・変数の読み取りを自然言語指示で行えます。

技術的な仕組み：
- **MCP（Model Context Protocol）**: AIエージェントが外部ツールを標準的な方法で操作するためのプロトコル。「AIエージェントが道具を使うための共通言語」
- **MATLAB MCP Server**: MATLABのセッションをMCPプロトコル経由で外部から制御するブリッジ
- **Cline**: MCPに対応したAIコーディングエージェント。ユーザーの自然言語指示を受け取り、MCPツール経由でMATLABを操作する

## 実装：ステップバイステップ

**前提条件**
- MATLAB R2024b 以降（`matlab.engine.shareEngine` が必要）
- VS Code 最新版
- Node.js 18 以上
- Cline VS Code 拡張: VS Code マーケットプレイスで "Cline" を検索してインストール
- MATLAB MCP Server: `npm install -g matlab-mcp-server`

**ステップ1: MATLABの共有セッション起動**

MATLABを起動し、以下を実行してエンジンを共有状態にします。

```matlab
% === ステップ1: MATLABエンジンを外部から接続可能にする ===
% ClineがMATLAB MCP Server経由でこのセッションに接続する
matlab.engine.shareEngine
disp('MATLABエンジン共有中 — Clineからの接続を待機中...')
```

**ステップ2: ClineのMCP設定ファイルを作成**

プロジェクトフォルダ直下に `.vscode/cline_mcp_config.json` を作成します。

```json
{
  "mcpServers": {
    "matlab": {
      "command": "matlab-mcp-server",
      "args": ["--shared-session"],
      "env": {}
    }
  }
}
```

**ステップ3: ダブルウィッシュボーンのロールセンター計算関数を用意**

チームのサスペンション設計フォルダに以下のMATLAB関数を置いておきます。

```matlab
% === ステップ3: ロールセンター高さ計算関数 ===
% calc_roll_center_height.m
% 入力:
%   camber_deg - キャンバー角 [°]（負値でネガティブキャンバー）
%   upper_pts  - アッパーアームの [内側; 外側] ハードポイント座標 [mm]
%   lower_pts  - ロワーアームの  [内側; 外側] ハードポイント座標 [mm]
%   track_half - トレッド幅の半分 [mm]
% 出力:
%   rc_height  - ロールセンター高さ [mm]（正 = 地上より上）

function rc_height = calc_roll_center_height(camber_deg, ...
    upper_pts, lower_pts, track_half)

% アームの傾き角（ラジアン）を計算
ua = atan2(upper_pts(2,2)-upper_pts(1,2), upper_pts(2,1)-upper_pts(1,1));
la = atan2(lower_pts(2,2)-lower_pts(1,2), lower_pts(2,1)-lower_pts(1,1));

% インスタントセンター（瞬間回転中心）= アーム延長線の交点
dx = upper_pts(1,1) - lower_pts(1,1);
dy = upper_pts(1,2) - lower_pts(1,2);
t  = (dy + dx*tan(la)) / (tan(ua) - tan(la) + 1e-12);
IC = upper_pts(1,:) + t * [cos(ua), sin(ua)];

% タイヤ接地点 → インスタントセンターを結ぶ線とX=0の交点がロールセンター
contact   = [track_half, 0];  % タイヤ接地点（トレッド半幅の位置）
slope     = (IC(2) - contact(2)) / (IC(1) - contact(1) + 1e-12);
rc_height = contact(2) - slope * contact(1);  % x=0での高さ
end
```

**ステップ4: Clineへの自然言語指示でパラメータスイープを実行**

VS Code上でClineを開き（⌘+Shift+P → "Cline: Open"）、以下のように入力します。

```
（Clineへの指示）
MATLABの calc_roll_center_height 関数を使い、
キャンバー角を -0.5° から -3.0° まで 0.5° 刻みでパラメータスイープして、
ロールセンター高さの変化をグラフと数値テーブルで出力してください。

ハードポイント座標（チームの実測値）:
  アッパーアーム内側: [195, 285]、外側: [598, 265]
  ロワーアーム内側:   [178, 195]、外側: [615, 182]
  トレッド半幅: 600 mm
```

Clineが自動生成・実行するスクリプト：

```matlab
% === Clineが自動生成したパラメータスイープスクリプト ===
% キャンバー角変化に対するロールセンター高さ分析

upper_pts  = [195, 285; 598, 265];  % 行1=内側, 行2=外側 [mm]
lower_pts  = [178, 195; 615, 182];
track_half = 600;

camber_range = -0.5 : -0.5 : -3.0;
rc_vals      = zeros(size(camber_range));

for k = 1:numel(camber_range)
    rc_vals(k) = calc_roll_center_height(camber_range(k), ...
        upper_pts, lower_pts, track_half);
end

% 結果テーブル表示
T = table(camber_range', rc_vals', ...
    'VariableNames', {'キャンバー角_deg', 'ロールセンター高さ_mm'});
disp('=== ロールセンター高さ スイープ結果 ===');
disp(T);

% グラフ描画（推奨範囲を赤・緑破線で表示）
figure('Name','キャンバー角 vs ロールセンター高さ');
plot(camber_range, rc_vals, 'b-o', 'LineWidth', 2.5, 'MarkerSize', 9);
xlabel('キャンバー角 [°]');  ylabel('ロールセンター高さ [mm]');
title('フロントサスペンション: キャンバー角 vs ロールセンター高さ');
yline(25, '--r', '下限 25mm');  yline(50, '--g', '上限 50mm');
grid on;
```

このコードを実行すると以下が出力されます：

```
=== ロールセンター高さ スイープ結果 ===
  キャンバー角_deg    ロールセンター高さ_mm
  _______________    __________________
       -0.5                  48.2
       -1.0                  43.5
       -1.5                  38.8
       -2.0                  34.1
       -2.5                  29.4
       -3.0                  24.8
```

グラフには許容範囲（25〜50mm）が破線で示され、-2.5° あたりが最適に近いことが視覚的に分かります。

## よくあるエラーと対処

| エラー | 原因 | 解決法 |
|--------|------|--------|
| `Cannot connect to MATLAB engine` | `shareEngine`未実行 | MATLABで `matlab.engine.shareEngine` を先に実行 |
| `matlab-mcp-server: not found` | npmグローバルインストール未完了 | `npm install -g matlab-mcp-server` を再実行し `npm root -g` でPATHを確認 |
| Clineが英語でコードを生成する | プロンプトが英語解釈される | 指示の冒頭に「日本語でコメントを入れてください」と追記 |
| `Function undefined: calc_roll_center_height` | MATLABパスが通っていない | MATLABで `addpath(genpath('/your/project'))` を実行 |

## 学生フォーミュラ・レース車両開発への応用

### 具体的なシナリオ

全日本学生フォーミュラ大会（8月開催）の2ヶ月前、鈴鹿サーキット南コース向けセットアップを検討している状況を想定します。設計担当の3年生が「キャンバー角を寝かせてコーナリンググリップを上げたいが、ロールセンターがどう変わるか計算できない」という状況です。

Cline + MATLAB MCPがあれば、自然言語で「キャンバーを-1.5°から-2.5°まで試した場合のロールセンター高さの変化を出して」と入力するだけで、5分後に数値とグラフが返ってきます。

### 背景理論の解説

**ロールセンター**（Roll Center）とは、コーナリング時に車体がロール（横揺れ）する際の瞬間的な回転中心です。地上からの高さが変わると、タイヤにかかる荷重移動の仕方が変わります。

- ロールセンターが**高い**：ロール角は小さくなるが、タイヤに「ジャッキング力」（コーナリング中にサスペンションが縮む・伸びる力）がかかりやすい
- ロールセンターが**低い**：ロールが大きくなるが、タイヤの動きがスムーズ

学生フォーミュラ規格の車両では、フロントのロールセンター高さは **25〜50mm** が一般的な目安とされています（車体設計・重心高に依存）。
