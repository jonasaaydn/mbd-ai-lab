## 学生フォーミュラにおける課題

学生フォーミュラの走行セッション後、エンジニアはMoTeC i2・AiM Race Studio・MSエンジニアリング等のDAQから出力された**20〜50チャンネル・数百秒分のCSVとグラフ**を短時間で解析しなければならない。しかし専任データエンジニアがいないチームでは、1走行セッションのデータ整理に**2〜3時間**かかることも珍しくない。

「どのコーナーでアンダーステア（タイヤの横力が足りず外側に膨らむ挙動）が出ているか」「スロットルオン点（ドライバーが再加速を開始する距離）が遅れているか」という問いに、次の走行までに答えを出せないケースが多発する。競技会の限られたガレージ時間では、この解析ボトルネックが直接ラップタイムに影響する。

---

## ChatGPTを使った解決アプローチ

GPT-4o（2024年5月〜）は画像・テキストを同時に処理できる**マルチモーダルLLM**（複数の入力形式を扱える大規模言語モデル）である。テレメトリソフトが出力したグラフのスクリーンショットをChatGPTにアップロードするだけで、波形の特徴・異常点・ドライバーのアクション解析を自然言語で得られる。

従来はデータエンジニアがグラフを目視確認して頭の中で解釈していた作業をAIが代行するため、**専門知識がないメンバーでも一次解析が可能**になる。特に「複数コーナーの同一チャンネル比較」や「ラップオーバーレイのズレ原因特定」でその効果が大きい。

## 実装：ステップバイステップ

**前提条件：**
- ChatGPT Plus（GPT-4o）またはChatGPT Free（GPT-4o mini）のアカウント
- MoTeC i2 / AiM Race Studio 等のテレメトリソフトで出力したグラフPNG/JPEG
- Python 3.10以上（複数グラフを自動化する場合）

### ステップ1：グラフ画像を直接アップロードして解析（5分で試せる）

ChatGPTのチャット画面でグラフ画像をドラッグ＆ドロップし、以下のプロンプトを送信する：

```
次のテレメトリグラフを解析してください。
車両：学生フォーミュラ（ターボエンジン、後輪駆動）
コース：鈴鹿サーキット　セクター1
チャンネル：縦軸=スロットル開度[%]、横軸=距離[m]

以下を答えてください：
1. スロットルオン点（再加速開始距離）を読み取る
2. パーシャルスロットル区間があれば箇所を指摘する
3. ヘアピン後の立ち上がり改善アドバイスを1つ提案する
```

**ChatGPTの出力例：**

```
グラフを確認しました。
・ヘアピン(約480m地点)でのスロットルオン点は510m付近です
・250m地点でスロットルが一度60%まで下がる
  パーシャルスロットル区間が200msほど見られます
・立ち上がりは520m〜560mで緩やかに開いており、
  ドライバーはトラクション不足を懸念しているようです
推奨：リアダウンフォースを5〜10%増加させると
スロットルオン点を5〜10m前進できる可能性があります
```

### ステップ2：Python APIで複数グラフを一括解析（自動化）

```python
# === 必要なライブラリのインストール ===
# pip install openai pillow pandas

import openai, base64, pandas as pd
from pathlib import Path

# === APIクライアントの初期化 ===
# 環境変数 OPENAI_API_KEY にキーをセットしておく
client = openai.OpenAI()

def encode_image(path: str) -> str:
    """画像をBase64エンコード（API送信用）"""
    with open(path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

def analyze_telemetry_graph(image_path: str, channel_info: str) -> str:
    """グラフ画像を送信してGPT-4oに解析させる"""
    img_b64 = encode_image(image_path)

    response = client.chat.completions.create(
        model="gpt-4o",          # マルチモーダル対応モデル
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:image/png;base64,{img_b64}",
                            "detail": "high"  # 高解像度モードで細部まで解析
                        }
                    },
                    {
                        "type": "text",
                        "text": f"""
学生フォーミュラのテレメトリグラフを解析してください。
チャンネル情報: {channel_info}

以下の形式でJSON出力してください:
{{
  "問題点": ["問題1", "問題2"],
  "推奨セットアップ変更": ["変更1", "変更2"],
  "ドライバーへのフィードバック": "フィードバック文",
  "優先度": "high/medium/low"
}}
                        """
                    }
                ]
            }
        ],
        max_tokens=800
    )
    return response.choices[0].message.content

# === セッション内の全グラフを一括処理 ===
graph_dir = Path("./telemetry_graphs")  # グラフPNGが入ったフォルダ

channel_configs = {
    "throttle.png": "縦軸=スロットル開度[%] 横軸=距離[m]",
    "steering.png": "縦軸=ステアリング角度[deg] 横軸=距離[m]",
    "speed.png":    "縦軸=車速[km/h] 横軸=時間[s]",
}

results = []
for filename, info in channel_configs.items():
    path = graph_dir / filename
    if path.exists():
        result = analyze_telemetry_graph(str(path), info)
        results.append({"ファイル": filename, "解析結果": result})
        print(f"✓ {filename} 解析完了")

# === 結果をCSVで保存 ===
df = pd.DataFrame(results)
df.to_csv("analysis_results.csv", index=False, encoding="utf-8-sig")
print("→ analysis_results.csv に保存完了")
```

**実行結果：**
```
✓ throttle.png 解析完了
✓ steering.png 解析完了
✓ speed.png 解析完了
→ analysis_results.csv に保存完了
```

## よくあるエラーと対処

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `invalid_image_url` | Base64変換ミス | `detail="auto"`に変更してリトライ |
| グラフの数値が読めない | 解像度不足（低DPI） | PNG 1920×1080以上・300DPI以上で出力 |
| JSONパースエラー | GPT-4oが自由文で返す | プロンプト末尾に「必ずJSONのみを返してください」を追記 |
| レート制限エラー | Tier 1は1分20リクエスト | `time.sleep(3)`を各リクエスト間に挿入 |
| 数値の読み取りが不正確 | X/Y軸ラベルが小さい | グラフ画像を1枚ずつ送信・軸情報をテキストで補完する |

## 学生フォーミュラ・レース車両開発への応用

### シナリオ：エンデュランスレース前日の総合データレビュー

学生フォーミュラのエンデュランス競技（22km走行）前日、チームは**複数ドライバーの走行データを比較して最終セットアップを決定**しなければならない。通常は5〜6名のエンジニアが数時間かけて行うこの作業を、ChatGPTのGPT-4o Visionで効率化する。

### 背景理論

GPT-4oのビジョン機能は、画像内のパターン認識・波形の特徴抽出・テキストとの統合解釈が得意である。テレメトリグラフは「時系列信号の視覚表現」であり、**ドメイン知識（レース工学）をプロンプトで補完**することで、モデルは専門家に近い解釈を生成できる。特に「複数グラフの比較」「正常波形と異常波形の差分検出」は、人間が疲弊する長時間セッション後でも精度が落ちない点が強みだ。

### 実際に動くコード

```python
# エンデュランス向け追加チャンネルを channel_configs に追加
channel_configs_endurance = {
    "fuel_injection.png": "縦軸=燃料噴射量[ms] 横軸=ラップ番号（燃費トレンド確認）",
    "water_temp.png":     "縦軸=水温[℃] 横軸=時間[s]（オーバーヒート監視）",
    "lambda.png":         "縦軸=空燃比λ 横軸=エンジン回転数[rpm]（λ=1.0付近が理想）",
    "lap_delta.png":      "縦軸=ラップタイムΔ[s] 横軸=ラップ番号（タイヤ劣化推定）",
}

# エンデュランス特化プロンプト
def analyze_endurance(image_path: str, channel: str) -> str:
    prompt = f"""
エンデュランスレース（22km）のデータです。チャンネル: {channel}
以下の観点で解析してください：
1. タイヤ劣化の兆候（ラップタイム悪化・コーナリング速度低下）
2. ドライバー疲労の兆候（スロットル・ブレーキ操作のバラつき増加）
3. 車両信頼性リスク（温度・圧力の上昇トレンド）
    """
    return analyze_telemetry_graph(image_path, channel)
```
