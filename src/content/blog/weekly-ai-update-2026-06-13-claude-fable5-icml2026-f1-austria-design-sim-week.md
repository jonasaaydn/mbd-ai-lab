## トピック1：Claude Fable 5リリース（6月9日）—— MBD/工学タスクへの影響

Anthropicが6月9日に公開した **Claude Fable 5** は、同社初の「Mythos級」モデルだ。1Mトークンのコンテキストウィンドウ（出力128Kトークン）を持ち、長時間にわたる自律エージェント作業のために設計されている。

### ベンチマーク上の位置づけ

Anthropicの公式発表によると、Fable 5は同社の複雑長時間分析タスクベンチマークで **90%超（前モデルOpus比+10ポイント）** を記録。Artificial Analysis社のIntelligence Indexでは **スコア65** を獲得し、OpenAI GPT-5.5（60）、Google Gemini 3.1 Pro Preview（57）を上回った。コーディングおよびエージェント的ツール使用ベンチマークでは **123モデル中2位** という結果だ。

### MBDエンジニアへの具体的な変化

Fable 5が特に優れているのは「タスクが長く・複雑になるほど他モデルとの差が広がる」点だ。MBD開発における実務シナリオで考えると：

| シナリオ | Opus 4.8との比較 |
|---------|----------------|
| Simulink大規模モデルの一括リファクタリング | コンテキスト長2倍→一括処理可能 |
| CFD解析レポートの自動生成 | 長文出力の品質が大幅向上 |
| ISO 26262要件トレーサビリティ生成 | 複雑な論証構造を一気通貫で出力 |
| MATLAB Agentic Toolkitとの連携 | 長時間エージェント実行の精度が向上 |

### APIアクセスと価格

Claude API（Anthropic）、Amazon Bedrock、Vertex AI、Microsoft Foundryで即日利用可能。価格は **入力$10/Mトークン、出力$50/Mトークン**（Opus 4.8の約2倍）。1Mトークンコンテキストを活かした長時間バッチ処理の場面では、Opus複数回呼び出しより費用対効果が高いケースも出てきている。

### 注意点

Anthropicは高リスク領域（サイバーセキュリティ・生物化学・核関連）でFable 5の応答をブロックし、Opus 4.8にフォールバックする安全設計を採用。MBD/CAEの通常業務への影響はないが、安全関連システムの設計支援を行う際はこの制限を事前に確認しておく必要がある。

## トピック2：ICML 2026 AI4Physicsワークショップ—— 採択論文プレビュー

ICML 2026（**7月6〜11日、韓国・ソウル**）の採択論文リストが先週末に公開された。全体では23,918件の投稿から **6,352件が採択**（採択率26.6%）という過去最大規模だ。

物理AI特化の **AI4Physicsワークショップ** も採択されており、テーマは「機械学習で物理発見を端から端まで支援する信頼できるAI手法」。具体的なフォーカスは：

- **神経演算子（Neural Operators）**：FNO・DeepONet・Poseidonなどの次世代手法
- **物理制約付き生成モデル**：設計空間をサンプリングしながら物理則を保持する
- **微分可能シミュレータ**：シミュレータを通じて逆伝播が可能な統合手法
- **ハイブリッドソルバー**：従来ソルバーとNNを組み合わせた実用手法
- **誤差制御付きエミュレーション**：信頼区間・不確実性定量化付きサロゲート

MBD/CAEエンジニアにとって特に注目すべきは「誤差制御付きエミュレーション」と「閉ループ実験設計（今週記事1本目のActive Learningも含む）」のセッション群だ。論文プレプリントはICML公式サイトおよびOpenReviewで順次公開されている。

### 注目トレンド：物理AI論文の質的変化

昨年のICML 2025と比較すると、2026年採択論文には「理論的証明だけでなく、実工学問題への定量的実証を含む」ものが増加した。CFD・FEAサロゲートモデルで「MAE 5%以下・計算コスト100倍削減」などの具体的なベンチマークが求められるようになってきている。

## トピック3：F1 2026シーズン前半でのAI実戦展開

2026年F1シーズンが大規模な技術規則変更（アクティブエアロ・ハイブリッド比率変更）とともに開幕し、AIシステムの戦略的役割が従来以上に複雑化している。

### FIA ECAT：AI搭載トラックリミット監視

FIAが2026年から導入した **ECAT（Every Car All Turns）** システムは、コンピュータビジョンとマイクロセクターデータを統合してトラック超過を自動検知するAIシステムだ。ケースの **最大95%** をソフトウェアが自動判定し、人間の審査に送るのは5%以下に絞られる。従来はカメラ映像を人間が確認していたが、AIにより「見落とし」と「主観的判定」の両方が解消されつつある。

### 天気AI：Tomorrow.io DeepSkyの全面展開

FIAと**Tomorrow.io**が契約したDeepSkyシステムが2026年全戦で稼働中だ。衛星AIで生成される超局所予測は、ピットウォールのポータルに直接配信され、レースコントロールの熱中症警告・雨宣言の公式判断基準となっている。カナダGPでは雨上がりのタイミング予測誤差が**3分以内**に収まり、複数チームのピット戦略判断に活用された。

### 次戦：オーストリアGP（6月27〜29日）に向けた技術動向

レッドブルリンクはカナダ・モントリオールと異なり「低ダウンフォース系ではなく中間ダウンフォース」設定が求められるサーキットだ。アクティブエアロの展開戦略がラップタイムに直結するため、各チームのAIシステムが最適エアロ展開シーケンスを今週末から本格シミュレーションしているとみられる。

## 学生フォーミュラ・レース車両開発への応用

### シナリオ：Claude Fable 5でSimulinkモデルをシーズン前に一括リファクタリング

大会シーズン前の「技術車検対策リファクタリング」はチームの共通課題だ。モデルの信号名不統一・未使用ブロック・コメント漏れを一括して修正したいが、数十個のサブシステムを手動でチェックするのは時間がかかる。

Fable 5の1Mトークンコンテキストを使うと、**Simulinkモデルのコード全体（自動生成Cコード＋MLXファイルのXML表現）を一括投入**して「CLAUDE.mdに定めたチーム規約に準拠しているか」をまとめてレビューさせることが可能になった。

### 実際に動くコード例

```python
# pip install anthropic
import anthropic
import pathlib

# === ステップ1: SimulinkモデルのXML表現を読み込む ===
# Simulinkモデルを "save_system('model','model.slx')" でslx化した後、
# slxはZIPアーカイブなのでunzipしてmodel.xmlを取得できる
model_xml = pathlib.Path("vehicle_dynamics.xml").read_text(encoding="utf-8")

# チームのコーディング規約（事前に作成しておく）
team_standard = pathlib.Path("team_simulink_standard.md").read_text(encoding="utf-8")

# === ステップ2: Claude Fable 5にレビューを依頼 ===
client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-fable-5",          # Mythos級モデルを使用
    max_tokens=8192,
    messages=[
        {
            "role": "user",
            "content": f"""以下のSimulinkモデルXMLと、チーム規約を読んで
規約違反箇所を一覧化してください。

## チーム規約
{team_standard}

## Simulinkモデル（XML）
{model_xml}

出力形式: Markdown表（ブロック名, 違反内容, 推奨修正）"""
        }
    ]
)

print(message.content[0].text)
```

**実行結果（例）:**

```markdown
| ブロック名 | 違反内容 | 推奨修正 |
|------------|---------|---------|
| SteerAngle_Gain | 信号名が英数字のみでチーム規約のスネークケース規則に違反 | steer_angle_gain に変更 |
| Subsystem_3 | サブシステム名が意味不明 | FrontWing_Aero_Calc など機能名に変更 |
| ...（計23件） | | |
```
