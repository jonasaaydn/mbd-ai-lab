## SimScale 2026 State of Engineering AI報告書：設計バリアントが4倍に

### 調査概要

2026年3月、SimScaleが発表した「**State of Engineering AI 2026**」報告書は、米国・英国・ドイツの工学リーダー350名を対象に実施したグローバル調査の結果だ（調査実施: 2026年2月、Global Surveyz社との共同実施）。

### 主要な数字

| 指標 | 従来手法 | AI活用ワークフロー | 改善率 |
|------|---------|------------------|--------|
| 1プログラムあたりの設計バリアント数 | 基準 | **4倍近く** | +300% |
| シミュレーション依頼の処理速度 | 基準 | **2.8倍** | +180% |
| RFQ・技術提案書の作成速度 | 基準 | **3倍** | +200% |

### 最重要な洞察

報告書の核心は「**AIは既存ワークフローを速くするのではなく、試せるアイデアの総量を増やす**」という点だ。従来は設計時間の制約から5〜10案しか評価できなかった設計が、AI活用チームでは20〜40案を同じ期間で検証できるようになった。

**開発早期での最適解発見**: バリアント数増加の真の価値は、開発末期の大規模変更を防ぐことにある。プロトタイプ後に問題を発見するコストは、設計初期の数十倍から百倍になる。

### 数字の意味するもの

学生フォーミュラチームにとって、「設計バリアント4倍」は特に重要だ。フロントウィングの翼型選定・サスペンションジオメトリ最適化・冷却系レイアウトなど、試したいが時間がなかった検討が、AIサロゲートの導入で一気に実行可能になる。

## Cadence × NVIDIA：sim-to-realギャップを埋める提携拡大

### 2026年4月の提携内容

CadenceとNVIDIAは2026年4月のCadenceLIVE Silicon Valley 2026で、**エージェンティックAI・物理シミュレーション・デジタルツイン**にわたる提携拡大を発表した。

**核心的な統合**:

これらを組み合わせることで、物理シミュレーション→大規模シナリオテスト→実世界フィードバック→ワールドモデル更新という**エンドツーエンドのエージェント型ワークフロー**が実現する。

### sim-to-realギャップとは

ロボティクスや自動運転で長年の課題だった「**シミュレーション内でうまく動くが実機では動かない**」問題。Cadenceの物理精度の高い多物理シミュレーターとNVIDIAのワールドモデル（Cosmos）を組み合わせて、シミュレーション環境と現実の差異（主にセンサーノイズ・材料非線形性・接触問題）を最小化する。

### モータースポーツへの応用

このアプローチはレース車両開発にも直接応用できる：

- **デジタルツイン精度向上**: Cadenceの熱・構造解析精度 × NVIDIAの大規模シナリオ生成で、デジタルツインが実車挙動をより正確に模擬
- **自律運転テスト**: フォーミュラEや将来の自動運転レーシングカーの開発で、実走行前にVTD環境で膨大なシナリオを事前評価
- **FMU連携**: Cadence-NVIDIA環境で生成したモデルをFMUとしてエクスポートしてSimulinkと連携する経路が開ける可能性

## 学生フォーミュラ・レース車両開発への応用

### シナリオ：SimScale式「設計バリアント4倍」を学生チームで再現する

SimScaleの報告書が示す「設計バリアント4倍」を、学生フォーミュラチームが無料ツールで実現するためのワークフローを紹介する。

#### 背景理論：DOE×AIサロゲートでバリアントを増やす仕組み

設計バリアント増加の鍵は「**AIサロゲートモデル（代替モデル）**」だ。1回のCFD計算に2〜4時間かかるとすると、10案の評価に最低20〜40時間必要になる。これをCFDの代わりに使える軽量AIモデルを最初に構築すれば、その後の評価は秒単位で実行できる。

**手順の概要**:
1. 設計パラメータを定義（翼型の迎角・キャンバー・スパン等）
2. 15〜30点のCFD計算でデータセット作成
3. ガウス過程回帰（GPR）または軽量ニューラルネットでサロゲートを構築
4. サロゲートで1000点以上の設計空間を探索

#### 実際に動くコード（フロントウィング設計最適化）

```python
import numpy as np
from sklearn.gaussian_process import GaussianProcessRegressor
from sklearn.gaussian_process.kernels import RBF, ConstantKernel
from scipy.optimize import differential_evolution

# === ステップ1: CFDデータを模擬（実際はOpenFOAMやSimScaleの結果を使う）===
# 設計パラメータ: [迎角[度], キャンバー比[-], スパン[mm]]
# 目標: ダウンフォース係数CLを最大化しつつ抗力係数CD/CLを最小化

np.random.seed(42)
# 15点のCFD計算結果（3変数の設計）
X_cfd = np.column_stack([
    np.random.uniform(3, 12, 15),    # 迎角 3〜12度
    np.random.uniform(0.05, 0.15, 15),  # キャンバー比
    np.random.uniform(800, 1200, 15)  # スパン [mm]
])

# 真のCFD応答を模擬する関数（実際はシミュレーション結果に置き換え）
def cfd_simulation(X):
    alpha, camber, span = X[:, 0], X[:, 1], X[:, 2]
    # ダウンフォース係数の近似式（実際はCFDから取得）
    CL = 0.8 * np.sin(np.deg2rad(alpha)) + 3.5 * camber + 0.0005 * (span - 1000)
    CD = 0.05 + 0.01 * alpha**1.5 + 0.3 * camber**2
    return CL / CD  # L/D比を最大化したい

y_cfd = cfd_simulation(X_cfd) + np.random.normal(0, 0.1, 15)  # 計測ノイズ

# === ステップ2: ガウス過程サロゲートを15点のCFDデータで学習 ===
kernel = ConstantKernel(1.0) * RBF(length_scale=[1.0, 0.05, 100.0])
gpr = GaussianProcessRegressor(kernel=kernel, alpha=0.1, n_restarts_optimizer=5)
gpr.fit(X_cfd, y_cfd)
print(f"サロゲート構築完了（15点のCFDデータを使用）")

# === ステップ3: サロゲートで1000点を瞬時に評価（CFDなら2000〜4000時間！）===
X_sweep = np.column_stack([
    np.random.uniform(3, 12, 1000),
    np.random.uniform(0.05, 0.15, 1000),
    np.random.uniform(800, 1200, 1000)
])
y_pred, y_std = gpr.predict(X_sweep, return_std=True)

# 最良設計を特定
best_idx = np.argmax(y_pred)
best_design = X_sweep[best_idx]

print(f"\n=== 最適設計（サロゲートによる1000案探索）===")
print(f"迎角:      {best_design[0]:.1f} 度")
print(f"キャンバー比: {best_design[1]:.3f}")
print(f"スパン:    {best_design[2]:.0f} mm")
print(f"予測L/D比: {y_pred[best_idx]:.2f} ± {y_std[best_idx]:.2f}")

# === ステップ4: この設計だけCFDで検証（計算時間: 2〜4時間×1回のみ）===
y_cfd_verify = cfd_simulation(best_design.reshape(1, -1))[0]
print(f"\nCFD検証結果: L/D = {y_cfd_verify:.2f}（サロゲート予測: {y_pred[best_idx]:.2f}）")
```
