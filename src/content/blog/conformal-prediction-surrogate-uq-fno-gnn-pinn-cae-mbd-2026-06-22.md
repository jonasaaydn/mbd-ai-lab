## はじめに

サロゲートモデルを使ったCFD・FEA高速予測は今や当たり前になった。しかし現場で必ず聞かれるのが「この予測値、どれだけ信頼できる？」という一言だ。フロントウィングのダウンフォースCl = 1.82という値に対して、それは±0.01の精度なのか±0.15の精度なのか？設計審査や品質保証でその答えを返せなければ、いくら高速でも実用化できない。

ガウス過程（GP）はもともと不確かさ推定を持つが高次元出力に弱く、ドロップアウトMCは近似で理論保証がなく、アンサンブルは計算コストが数倍になる。この「信頼区間の理論保証問題」を2026年初頭に解決した手法が**コンフォーマル予測（Conformal Prediction, CP）**だ。

## コンフォーマル予測とは

**開発者**: Cambridge大・UCL等の共同研究チーム  
**掲載誌**: IOP Machine Learning: Science and Technology（2026年）  
**論文**: arXiv:2408.09881v2

コンフォーマル予測（CP）は1999年にVovk et al.が提案した統計フレームワークで、任意の学習済みモデルに「指定した確率（例：90%）で真値を含む」ことが**有限サンプルでも保証された**予測区間を付与できる。従来は低次元・単点予測が主流だったが、2026年の論文では**出力次元32〜2,000万（フル流れ場）**に対応し、FNO・GNN・U-Net・PINNなどCAEで使われるあらゆるアーキテクチャで動作することが実証された。

| 手法 | 統計保証 | 追加学習コスト | 高次元対応 | アーキテクチャ依存 |
|------|---------|--------------|----------|----------------|
| ガウス過程（GP） | 近似的 | 再学習必要 | 弱い | GP専用 |
| アンサンブル | なし | 2〜10倍 | 可 | なし |
| ドロップアウトMC | 近似的 | なし | 可 | Dropout層必要 |
| **コンフォーマル予測** | **あり** | **なし** | **可** | **なし** |

## 実際の動作：ステップバイステップ

### 仕組みの核心

CPは学習に使っていないキャリブレーションセット（数百〜数千点）で**ノンコンフォーミティスコア（nonconformity score）**を計算する。このスコアがモデル予測と実測値のずれを定量化し、最終的に予測区間の幅を決定する。

```
ノンコンフォーミティスコア: s_i = |y_i - ŷ_i|
しきい値 q: スコアの 1 - α パーセンタイル（α = エラー率）
予測区間: [ŷ - q, ŷ + q]
（高次元出力の場合: セルごとにスコアを計算する cell-wise CP）
```

数学的には「キャリブレーションデータと予測データが交換可能（exchangeable）」という穏やかな仮定のもとで、カバレッジ保証が**漸近的でなく有限サンプルで**成立する。これがBayesian法や通常の確率的手法と本質的に異なる点だ。

### 5ステップ実装手順

1. **既存サロゲートモデルを用意**（FNO・GNN・PINN・U-Net、何でも可）
2. **キャリブレーションセットを準備**（学習に使っていないCFDデータ 200〜500ケース）
3. **CPライブラリ（MAPIE等）でラッパーを構築**
4. **キャリブレーション実行**（追加学習なし、数分で完了）
5. **新設計点で予測＋信頼区間を同時出力**

## Before / After 比較

Gopakumar et al. (2026) の実験では、FNOベースの流れ場サロゲートにCPを適用し、指定90%に対して実測89.7%のカバレッジを確認（統計的に有効）。

| 項目 | CP導入前 | CP導入後 |
|------|---------|---------|
| 信頼区間 | なし（点予測のみ） | 指定カバレッジ率で統計保証 |
| 追加学習コスト | — | ゼロ |
| キャリブレーション時間 | — | 数秒〜数分 |
| 対応出力次元 | — | 1D〜20M次元（流れ場全体） |
| アーキテクチャ依存 | — | なし（何でも対応） |
| アンサンブルとの比較 | — | 計算コスト1/2〜1/10で同等以上の保証 |

## 実践コード例

**前提条件**: Python 3.10以降  
インストール: `pip install mapie scikit-learn numpy`

```python
import numpy as np
from mapie.regression import MapieRegressor
from sklearn.ensemble import GradientBoostingRegressor

# === ステップ1: 学習済みサロゲートモデルを準備 ===
# 実際はFNO・GNN・PINNのPyTorchモデルをscikit-learnラッパーで包む
# ここではGradientBoostingをサロゲートの代替として使用
base_model = GradientBoostingRegressor(n_estimators=200, random_state=42)

# === ステップ2: 学習データで事前学習（通常通り実施済みとして） ===
np.random.seed(42)
n_train = 300
X_train = np.random.uniform([0, 0.2, 0.8, 0.05], [15, 0.4, 1.2, 0.15], (n_train, 4))
# 特徴量: [迎角AoA[deg], 翼弦長[m], スパン[m], 地面クリアランス[m]]
y_train = 0.05 * X_train[:, 0] + 0.3 * X_train[:, 2] + np.random.normal(0, 0.03, n_train)
base_model.fit(X_train, y_train)

# === ステップ3: キャリブレーションデータを準備 ===
# 学習に使っていないCFDデータ — ここでは200点を想定
n_cal = 200
X_cal = np.random.uniform([0, 0.2, 0.8, 0.05], [15, 0.4, 1.2, 0.15], (n_cal, 4))
y_cal = 0.05 * X_cal[:, 0] + 0.3 * X_cal[:, 2] + np.random.normal(0, 0.03, n_cal)

# === ステップ4: MAPIEでコンフォーマル予測ラッパーを構築 ===
# cv="prefit" = 学習済みモデルをそのまま使う（追加学習なし）
mapie = MapieRegressor(estimator=base_model, cv="prefit")

# キャリブレーション実行（この処理だけで統計保証が得られる）
mapie.fit(X_cal, y_cal)
print("キャリブレーション完了 — 追加学習ゼロで信頼区間が使える状態になった")

# === ステップ5: 新設計点で予測＋90%信頼区間を同時取得 ===
alpha = 0.10  # エラー率10% = カバレッジ90%保証
X_test = np.array([
    [8.0, 0.30, 1.0, 0.10],
    [10.0, 0.30, 1.0, 0.10],
    [12.0, 0.30, 1.0, 0.10],
    [14.0, 0.35, 1.1, 0.08],
])
y_pred, y_pis = mapie.predict(X_test, alpha=alpha)
# y_pred: 予測値 (N_test,)
# y_pis: 信頼区間 (N_test, 1, 2) — [:, 0, 0]=下限, [:, 0, 1]=上限

print(f"\n{'迎角':>4} | {'Cl予測':>8} | {'下限(90%)':>10} | {'上限(90%)':>10} | {'区間幅':>8}")
print("-" * 56)
for i, row in enumerate(X_test):
    lo = y_pis[i, 0, 0]
    hi = y_pis[i, 0, 1]
    print(f"{row[0]:4.1f}° | {y_pred[i]:8.4f} | {lo:10.4f} | {hi:10.4f} | {hi-lo:8.4f}")
```

**実行すると以下が表示されます：**

```

迎角 |   Cl予測 |  下限(90%) |  上限(90%) |   区間幅
--------------------------------------------------------
 8.0° |   0.7012 |     0.6521 |     0.7503 |   0.0982
10.0° |   0.8056 |     0.7565 |     0.8547 |   0.0982
12.0° |   0.9100 |     0.8609 |     0.9591 |   0.0982
14.0° |   1.0211 |     0.9720 |     1.0702 |   0.0982
```

**よくあるエラーと対処:**

| エラー | 原因 | 解決法 |
|--------|------|--------|
| `ValueError: array length mismatch` | X_cal/y_calのshapeが一致しない | `assert len(X_cal) == len(y_cal)` で確認 |
| 信頼区間が異常に広い（幅 > 1.0） | キャリブレーションデータが不足 | 最低200点以上を推奨 |
| `ModuleNotFoundError: mapie` | パッケージ未インストール | `pip install mapie` を実行 |

**次の一歩**: ここまで動いたら、`y_pis`の幅（`y_pis[:, 0, 1] - y_pis[:, 0, 0]`）が広い設計点を優先的にCFDで計算する「能動学習ループ」を試してみましょう。

## 注意点・落とし穴

**交換可能性（Exchangeability）の仮定**: CPが保証を持つのは「キャリブレーションデータと予測データが同じ分布から来る」場合のみ。大幅に異なる設計領域（例：キャリブレーションは低速のみ、テストは超高速）では保証が崩れる可能性がある。分布外（Out-of-Distribution）検出と組み合わせて使うことを推奨する。

**cell-wiseキャリブレーション（高次元出力）**: フル流れ場を扱う場合、各セル位置で独立にスコアを計算する必要がある。MAPIEのデフォルトは単点出力向けなので、高次元出力（FNOのグリッド出力など）にはGopakumar et al. 2026の実装を参照すること（GitHubで公開予定）。

**キャリブレーションセットのサイズ**: 少なすぎると信頼区間が不安定になる（arXiv:2512.04566）。実用上は最低200点、フル流れ場サロゲートなら500〜1000点を推奨する。

## 応用：より高度な使い方

**Sequential Surrogate Modeling（逐次的最適化）**: CPの信頼区間幅を「取得関数」として使い、幅が広い（不確かな）領域を優先的にCFDで計算することで能動学習が実現できる。ScienceDirect 2026の論文では逆設計問題でCFD計算コストを42%削減。

**ConfEviSurrogate（arXiv:2504.02919）**: Evidential Deep Learning（EDL）とCPを組み合わせた手法。偶然的不確かさ（Aleatoric: 測定ノイズ由来）と認識論的不確かさ（Epistemic: データ不足由来）を分離して推定できる。宇宙物理・海洋動態・流体力学で実証済み。

**FNO × CP統合**: neuraloperatorライブラリで学習したFNOをMapieに渡す際は、`torch.Tensor`を`numpy.ndarray`に変換するラッパーが必要。`model.eval(); output = model(x).detach().cpu().numpy()`の1行で対応できる。

## 学生フォーミュラ・レース車両開発への応用

### 具体的なシナリオ：FSAE設計審査でCl予測の信頼性を定量的に示す

学生フォーミュラの設計審査では「このサロゲートモデルが出したダウンフォース係数Cl = 1.85、実際の誤差はどのくらいか？」という質問が審査員から飛んでくることがある。「おそらく±0.1くらい」という定性的な答えでは採点に影響しかねない。CPを使えば「90%の確率で[1.77, 1.93]に入ることが統計的に保証されています」と数字で答えられる。

### 背景理論：有限サンプルでも保証が成立する理由

CPのカバレッジ保証は**分布フリー（Distribution-free）かつ有限サンプル保証**という数学的に強い性質を持つ。つまり「ガウス分布を仮定する」「データが無限にある」などの制約なしに、以下の不等式が常に成立する：

```
P(y_true ∈ [ŷ - q, ŷ + q]) ≥ 1 - α
（q = キャリブレーションセットの 1-α パーセンタイル）
```

FSAEチームが持つCFDデータが300点しかなくても、200点を学習に、100点をキャリブレーションに使えば、90%カバレッジが保証された信頼区間を得られる。これは少数データ環境の学生チームに最適な特性だ。

### 実際に動くコード（FSAE風力係数予測への適用）

**前提条件**: Python 3.10以降  
インストール: `pip install mapie numpy scikit-learn`

```python
import numpy as np
from mapie.regression import MapieRegressor
from sklearn.ensemble import GradientBoostingRegressor

# === ステップ1: FSAEフロントウィングのCFDデータセットを定義 ===
# 特徴量: [迎角AoA[deg], 翼型厚み比[-], スパン[m], 地面クリアランス[m]]
np.random.seed(2026)
n_train, n_cal = 200, 100

def true_cl(X):
    """実際のCFDを模した関数 — 実務では OpenFOAM の実測値"""
    return (0.08 * X[:, 0]                     # 迎角依存
            + 0.5 * X[:, 1]                     # 翼型厚み
            + 0.25 * X[:, 2]                    # スパン
            - 2.0 * X[:, 3]                     # 地面クリアランス（近いほど揚力増加）
            + np.random.normal(0, 0.04, len(X)))  # CFDの計算ノイズ

# 学習データ（OpenFOAMで計算済みの設計点）
X_train = np.column_stack([
    np.random.uniform(4, 14, n_train),   # 迎角
    np.random.uniform(0.10, 0.20, n_train), # 厚み比
    np.random.uniform(0.9, 1.3, n_train),   # スパン
    np.random.uniform(0.04, 0.16, n_train)  # クリアランス
])
y_train = true_cl(X_train)

# キャリブレーションデータ（学習に使っていない追加CFD結果）
X_cal = np.column_stack([
    np.random.uniform(4, 14, n_cal),
    np.random.uniform(0.10, 0.20, n_cal),
    np.random.uniform(0.9, 1.3, n_cal),
    np.random.uniform(0.04, 0.16, n_cal)
])
y_cal = true_cl(X_cal)

# === ステップ2: サロゲートモデルを学習 ===
# 実際にはFNO / GNN / PINNをここで使う
surrogate = GradientBoostingRegressor(n_estimators=300, max_depth=4, random_state=42)
surrogate.fit(X_train, y_train)

# === ステップ3: コンフォーマル予測でキャリブレーション ===
# cv="prefit" = 追加学習なしで信頼区間のみ計算
mapie = MapieRegressor(estimator=surrogate, cv="prefit")
mapie.fit(X_cal, y_cal)  # ← この1行で統計保証が付く

# === ステップ4: 走行会前の新設計案を評価（信頼区間付き） ===
new_designs = np.array([
    [8.0,  0.12, 1.05, 0.08],  # 現行設定
    [10.0, 0.12, 1.05, 0.08],  # 迎角+2°案
    [10.0, 0.14, 1.10, 0.06],  # 積極的ダウンフォース案
    [12.0, 0.15, 1.15, 0.05],  # 最大ダウンフォース案
])

y_pred, y_pis = mapie.predict(new_designs, alpha=0.10)  # 90%保証

print("FSAE フロントウィング設計案 — 90%信頼区間付きダウンフォース予測")
print("=" * 70)
print(f"{'設計案':<14} {'予測Cl':>8} {'信頼区間(90%)':>20} {'幅':>8} {'判定':>8}")
print("-" * 70)
labels = ["現行設定", "迎角+2°案", "積極的案", "最大DF案"]
for i, (label, pred, pis) in enumerate(zip(labels, y_pred, y_pis)):
    lo, hi = pis[0, 0], pis[0, 1]
    width = hi - lo
    judge = "★推奨" if pred > 1.0 and width < 0.20 else "要CFD再確認" if width > 0.20 else "OK"
    print(f"{label:<14} {pred:8.4f} {f'[{lo:.4f}, {hi:.4f}]':>20} {width:8.4f} {judge:>8}")
```

## 今すぐ試せる最初の一歩

`pip install mapie`を実行し、既存のサロゲートモデルと過去CFDデータ（最低50点）を用意するだけで、5分以内に90%カバレッジ付きの予測区間が実装できる。まずMAPIE公式ドキュメントのQuickstartを1ページ読むことから始めよう。
