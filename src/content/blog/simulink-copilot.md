## Simulink Copilotとは

MATLAB R2024b以降に搭載されたAI支援機能です。Simulinkのチャットウィンドウに自然言語で指示を入力すると、ブロックの追加・配線・パラメータ設定・モデルの説明生成などを実行します。

2026年現在、以下の3機能が実用段階に達しています。

| 機能 | できること | 実用度 |
|------|-----------|--------|
| ブロック生成 | 説明→ブロック追加・配線 | ★★★★☆ |
| モデル説明 | 既存モデルを日本語で解説 | ★★★★★ |
| デバッグ支援 | エラー原因の特定と修正提案 | ★★★☆☆ |

## 実例1：ルックアップテーブルの追加と配線

### やりたいこと

エンジン回転数（rpm）を入力にして、キャリブレーションデータからトルク（Nm）を出力する1-Dルックアップテーブルを追加したい。

### Copilotへの入力

```
エンジン回転数を入力としてトルクを出力する
1-D Lookup Table ブロックを追加してください。
rpm軸は [0, 1000, 2000, 3000, 4000, 5000, 6000]、
トルク値は [0, 45, 120, 180, 210, 195, 160] です。
既存の Engine_Speed 信号から接続してください。
```

### Copilotの動作

1. `1-D Lookup Table` ブロックを自動配置
2. Breakpoint と Table data を指定値で自動設定
3. `Engine_Speed` 信号から入力ポートへ自動配線
4. 出力ポートに `Engine_Torque` というラベルを追加

**所要時間：手作業の約5分 → Copilot使用で約40秒**

ブロックを探してドラッグして、パラメータダイアログを開いて数値を入力して配線する——この一連の操作がほぼゼロになります。

## 実例2：シミュレーションエラーのデバッグ

### 状況

シミュレーションを実行したら以下のエラーが出た：

```
Error in 'SuspensionModel/DampingForceCalc':
Algebraic loop detected.
Output signal 'damping_force' feeds back into input
through 'VehicleBody/FrontAxle'.
```

### Copilotへのエラー貼り付け

エラーメッセージをそのままCopilotのチャットに貼り付けると：

```
[Copilot回答]
代数ループが検出されています。原因と対策を説明します。

【原因】
DampingForceCalc の出力 damping_force が
FrontAxle を経由して自分自身の入力に戻っています。
これは同一タイムステップ内で解が確定しない状態です。

【対策案1（推奨）】
ループ内に Unit Delay ブロックを挿入して
1タイムステップの遅延を追加します。
→ Ts=0.001s（1kHz制御の場合）を設定してください。

【対策案2】
Algebraic Constraint ブロックで代数方程式として定式化する。
ただし、この場合はソルバーをodeXXに変更する必要があります。

対策1を自動で適用しますか？
```

**「はい」と入力すると、Copilotが Unit Delay を自動挿入して再配線します。**

## 実例3：デザインレビュー用のモデル説明生成

サスペンション制御モデルのサブシステムを選択して「このサブシステムを日本語で説明して」と入力すると：

```
[Copilot生成テキスト]

## SuspensionController サブシステム

本サブシステムは車体速度・ヨーレート・ステアリング角を入力として
ダンパー推力指令を生成するPID制御器です。

**制御ロジック**
- 高速域（>120km/h）：安定性優先でKpを0.7倍に低減
- 旋回時（|ヨーレート|>0.3rad/s）：ロール抑制ゲインを有効化
- フェイルセーフ：入力信号断線時はデフォルト推力500Nに固定

**主要パラメータ**
Kp=2.5, Ki=0.8, Kd=0.3（MATLAB Workspace変数から参照）
```

このテキストをそのまま仕様書のドラフトとして使えます。**4時間かかっていたドキュメント作業が30分に短縮**された事例があります。

## 現実的な限界：こんな用途は苦手

- **複雑な非線形制御器の一からの設計**：スケルトンを作らせて人間が詰める使い方が現実的
- **AUTOSAR準拠モデルの自動生成**：コーディング規準への対応は別途確認が必要
- **1000ブロック超の大規模モデル操作**：指示が複雑になると意図通りに動かないことがある

## 今すぐ試す：最初の5分

MATLAB R2024b以降をお持ちであれば：

1. Simulinkを開き、任意のモデルを表示
2. 右下の **「Copilot」パネル** をクリック
3. 以下を入力してみる：

```
このモデルに Gain ブロックを追加して、
値を 0.5 に設定してください
```

これだけで体験できます。まず小さなタスクで感触を掴んでから、実際の開発モデルに適用するのがおすすめです。
