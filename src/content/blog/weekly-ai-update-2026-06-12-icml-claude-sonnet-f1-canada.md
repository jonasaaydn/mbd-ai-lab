## 1. ICML 2026：物理AI論文が史上最多採択——SSM・拡散モデル・GNNが三つ巴

国際機械学習会議ICML 2026（バルセロナ、7月13〜19日）の採択通知が6月第2週に完了した。「Physics AI」タグを持つ採択論文は前年比**240%増**に達したことが査読者SNSの集計から浮上した（公式統計は7月本会議で発表予定）。

**MBD/CAE分野への直撃テーマ別採択数（推定）：**

| テーマ | 採択数（推定） | 筆頭機関 |
|--------|-------------|---------|
| Mamba/SSM × 物理シミュレーション | 8件 | MIT, ETH Zürich, Stanford |
| Physics-informed 拡散モデル | 12件 | CMU, KAIST, TU Delft |
| マルチフィデリティニューラルオペレータ | 5件 | Caltech, DeepMind |
| GNN × 構造・流体 FEA/CFD | 7件 | TU Munich, Imperial |
| 自動車・航空テレメトリへのSSM適用 | 3件 | MIT×McLaren, Toyota Research |

特に注目が集まっているのが MIT と McLaren の共著論文「**MMamba-Race: Multi-Modal State Space Models for Real-Time Motorsport Telemetry Prediction**」だ。レース車両テレメトリ（200Hz・20チャンネル）にMamba2アーキテクチャを適用し、Transformer比で**推論8倍速・RMSE 23%改善**を達成。論文コードはICML開催後にGitHubで公開予定とされており、FSAEチームが追実装できるレベルの実装が含まれる見込みだ。

**エンジニアへの影響：** ICML採択論文は通常、アクセプト後数ヶ月でPyPIパッケージ化・HuggingFaceモデル公開まで進む。今回の物理AI採択ラッシュを受け、今年後半には「インストール一発で使えるCAE向けMambaツール」が複数登場すると予想される。

## 2. Claude Sonnet 4.6：工学推論のベンチマークを静かに更新

Anthropicが6月10日に公開した内部評価レポートによると、Claude Sonnet 4.6（現在の最新Sonnetモデル）は工学・科学分野の推論タスクで前世代比**28%向上**を記録した。

**MBDエンジニアに関係する具体的な改善点：**

**① MATLAB/Simulinkコード生成精度の向上**  
構文エラー率が前世代の12%から4%に低下。特に Signal Builder・Rate Transition・Algebraic Loop を含む複雑なモデルでの生成品質が改善されている。

**② 物理的整合性チェックの強化**  
「このPIDゲインは安定か」「ここで次元が合わないのでは」といった物理的矛盾を指摘する能力が強化された。数式の符号ミス（エネルギー保存・モーメント方向など）の自動検出率が上昇している。

**③ 長文コンテキストでの大規模モデル解析**  
200Kトークン枠内での大規模Simulinkモデルのコード解析が安定化した。数十ファイルにわたるモデル構成を一括して把握し、不整合箇所を特定できるようになった。

```matlab
% 【新機能】Claude Sonnet 4.6 への新プロンプト例
% 「この制御則の安定性を根拠付きで説明してください」

% 前世代の回答: 「安定していると思われます。」
% Sonnet 4.6 の回答例（実際の応答から抜粋）:
%
% 「システム行列 A = [-2, 1; 0, -3] の固有値を計算します：
%  det(A - λI) = (-2-λ)(-3-λ) - 0 = λ² + 5λ + 6 = 0
%  → λ₁ = -2, λ₂ = -3
%  両固有値の実部が負のためリャプノフ安定条件を満たします。
%  最も収束が遅い固有値は λ₁ = -2 なので
%  収束時定数は τ = 1/|Re(λ₁)| = 0.5秒と推定できます。
%  ステップ応答は約 2τ = 1秒で定常値の 86.5% に到達します。」

% → 数式付き・数値付きで根拠が示される点が新しい
```

Claude Code + MATLAB Agentic Toolkit の組み合わせで「なぜそのパラメータ設定なのか」という設計根拠を数式付きで記録するワークフローが、MBDチームの設計レビュー資料作成を大幅に効率化している。

## 3. F1カナダGP 2026：AI戦略システムが「天候適応」機能で実戦進化

第8戦カナダGP（Circuit Gilles Villeneuve、6月14〜16日）に向けて、複数チームが「リアルタイム気象AIと統合したピット戦略システム」を本格投入することを明らかにした。

**Oracle Red Bull Racing**  
シーズン前から稼働中の40億シミュレーションベース戦略AIに、Tomorrow.io DeepSkyの局地気象予測（精度±0.3mm/h）が統合された。前戦スペインGPと前々戦モナコGPでは**最終ラップの天候急変を8〜12分前に予測**し、タイヤ交換タイミングを自動修正するシステムが機能していたことを首席ストラテジストが確認した。

**Mercedes-AMG Petronas**  
Mercedes-TU MunichのReq2Road研究チームと共同で開発した「戦略シナリオトレーサビリティAI」が初投入予定。過去5年間のモントリオールGPデータを基に、セーフティカー発生条件（周回数・タイヤ劣化度・天候・インシデント確率）を確率分布として可視化するダッシュボードをピットウォールに展開する。

**カナダGPがAI検証に適した理由**  
モントリオールは壁に近いレイアウトとカナダ特有の激しい天候変化が組み合わさるため、状況変化が多く**AIの動的意思決定能力の優位性が最も出やすいサーキット**として知られている。2025年カナダGPでは、AI推奨と異なる判断をした4チームすべてが最終的に順位を落としたことが後日データ分析で判明し、チーム内での「AI権限拡大」議論を加速させた。

## 4. MathWorks R2026b プレビュー：Agentic Toolkit 2.0 が全製品展開

MathWorksが6月11日に公開した R2026b Early Access Preview で、**Simulink Agentic Toolkit 2.0** の全製品統合が明らかになった。正式リリースは2026年9月を予定。

**主な新機能：**

**① Cross-Product MCP（全ツール統合）**  
Simulink・Polyspace・Requirements Toolbox・Stateflow・Signal Processing Toolboxが**単一の MCP サーバー**に統合された。AIエージェント（Claude Code・GitHub Copilotなど）がツール間を横断して「要件 → モデル → コード → テスト → 認証」まで一気通貫で自動生成できる環境が整う。

**② Vision-Language インターフェース**  
Simulinkブロック線図のスクリーンショットを添付するだけで「このモデルに何が足りないか」「ここの信号経路が正しいか」を自動分析する機能が試験搭載された。設計レビュー会議での活用が想定されている。

**③ Live Validation（試験機能）**  
AIが生成したモデルをリアルタイムでISO 26262 ASIL-B相当の形式検証にかける機能が追加。モデル生成→安全解析→修正→再生成のループが自動化される。

## 学生フォーミュラ・レース車両開発への応用

### シナリオ：ICML 2026採択論文を即日追実装してチームの技術力を示す

学生フォーミュラチームが「最先端論文を実装した」という実績は、スポンサー獲得と技術報告書の両面で競合チームとの差別化になる。ICML 2026の採択論文コードはOpenReviewで公開されており、採択通知から数日で動作可能なコードが入手できる。

**背景理論：**  
ICML（International Conference on Machine Learning）は世界最高峰の機械学習会議で、採択率は通常20〜25%だ。「論文採択＝実装・再現性が検証済み」を意味するため、査読なし論文（arXiv投稿のみ）より信頼性が高い。物理AIカテゴリが240%増採択という事実は「機械学習コミュニティが物理シミュレーション問題を本気で解き始めた」ことを示す転換点だ。

**今週の採択論文を活用する手順：**

```bash
# === ステップ1: ICML 2026の物理AI論文を探す ===
# OpenReviewで検索（査読コメントも公開されている）
# https://openreview.net/  → ICML 2026 で検索
# キーワード: "motorsport", "CFD", "surrogate", "telemetry", "vehicle dynamics"

# === ステップ2: 論文のコードリポジトリをクローン ===
# 採択論文のAbstractページ → "Code" リンクをクリック
git clone <論文のGitHubリポジトリURL>
cd <リポジトリ名>

# === ステップ3: 依存関係インストールと動作確認 ===
pip install -r requirements.txt
python demo.py  # 著者提供のデモを最初に動かす

# === ステップ4: 自チームのFSAEデータで実行 ===
# 通常はconfig.yamlまたはargs.pyでデータパスを変更するだけ
# data_path: "./fsae_2026/test_day_01/"
# channels: ["speed_kmh", "throttle_pct", "brake_pct", "steer_deg"]
python train.py --config configs/fsae.yaml
```

| テレメトリ予測精度（RMSE）| 0.042（商用ツール標準）| 0.028（33%向上）|
| 技術報告書での差別化ポイント | ツール活用事例のみ | 最先端研究の実装・検証結果 |
| スポンサー提案時の説得力 | 「AIを使っています」| 「ICML採択手法を自チームで実装・改良しました」|
| 設計サイクル（解析→改善策）| 翌日 | 当日15分以内 |
