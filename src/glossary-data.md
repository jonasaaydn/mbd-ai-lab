### MBD（モデルベース開発 / Model-Based Development）
制御システムやアルゴリズムを、いきなりコードで書くのではなく、まず Simulink などで「モデル（ブロック線図）」として作り、シミュレーションで検証してから実装する開発手法。手戻りを減らせるため自動車・レース車両開発の標準になっています。

### サロゲートモデル（代理モデル / Surrogate Model）
時間のかかるCFDやFEA解析の入力と出力の関係を、機械学習で近似した「身代わりモデル」。一度学習させれば、数時間かかる解析を数秒で予測できます。学生フォーミュラでは形状最適化の試行回数を一気に増やせます。

### PINN（物理情報ニューラルネットワーク / Physics-Informed Neural Network）
ニューラルネットワークの学習時に、支配方程式（流体ならNavier-Stokes式など）を制約として組み込む手法。少ないデータでも物理的に正しい予測ができ、温度分布や流れ場の予測に使われます。

### CFD（数値流体力学 / Computational Fluid Dynamics）
空気や水の流れをコンピュータで計算する解析。ウィングのダウンフォースや抗力を評価できますが、1ケース数時間かかるのが課題で、AIサロゲートによる高速化が進んでいます。

### FEM / FEA（有限要素法 / 有限要素解析）
構造物を細かい要素に分割し、応力や変形を計算する手法（FEMが手法、FEAが解析作業）。アップライトやサスペンションアームの強度・軽量化設計に使います。

### スリップ率（Slip Ratio）
タイヤの周速度と車体速度の差の割合。0なら完全グリップ、大きいほど空転を表します。一般に10〜15%付近でグリップが最大になり、トラクションコントロールはこの値を目標に制御します。

### Pacejka モデル（マジックフォーミュラ / Magic Formula）
スリップ率・スリップ角とタイヤ発生力の非線形な関係を、少数の係数で表す経験式。車両運動シミュレーションのタイヤモデルとして広く使われ、走行データから係数を同定します。

### ヨーレート（Yaw Rate）
車両が鉛直軸まわりに回転する角速度（旋回の速さ）。車両の安定性・回頭性の指標で、ヨーレート制御やMPCの制御対象になります。

### PID制御
誤差に対して比例（P）・積分（I）・微分（D）の3要素で操作量を決める、最も基本的なフィードバック制御。トラクションコントロールや温度制御など幅広く使われます。

### MPC（モデル予測制御 / Model Predictive Control）
システムの未来の挙動をモデルで予測しながら、最適な操作量を逐次計算する制御手法。PIDより高度で、制約（出力上限など）を扱えるのが強み。ヨーレート制御やエネルギーマネジメントに使われます。

### FMU / FMI（協調シミュレーション）
異なるツールで作ったモデル同士を繋いでシミュレーションするための標準規格（FMIが規格、FMUがその部品）。Simulinkの制御モデルとGT-SUITEのエンジンモデルを連成させる、といった使い方をします。

### トポロジー最適化（Topology Optimization）
与えられた荷重条件で、材料を「どこに残しどこを削るか」を最適化し、軽くて強い形状を自動生成する手法。AIと組み合わせてアップライトなどの大幅な軽量化に使われます。

### ベイズ最適化（Bayesian Optimization）
試行回数が限られる中で、次にどの設計点を試すと最も効率的かを確率的に選ぶ最適化手法。CFD/FEAのように1回が高コストな評価のチューニングに向いています。

### 転移学習（Transfer Learning）
大規模データで学習済みのモデルを、手元の少ないデータで再調整して使う手法。公開CFDデータセットで学習したモデルを自チームの形状に適用し、計算量を大幅に削減できます。

### ONNX（Open Neural Network Exchange）
学習済みAIモデルをツール間でやり取りするための共通フォーマット。PythonやMATLABで作ったモデルをECUや組込み環境に展開する橋渡しに使います。

### MCP（Model Context Protocol）
AIエージェント（Claude等）が外部ツールを操作するための共通規格。CursorやClaude CodeからMATLAB/Simulinkを直接動かす連携の土台になっています。

### SIL（Software-in-the-Loop）
制御ソフトを実機ECUではなくPC上で実行して検証するテスト手法。実車前にバグを洗い出せます（実機を使うHILと対比されます）。

### DoE（実験計画法 / Design of Experiments）
少ない試験回数で最大の情報を得られるよう、評価する条件の組み合わせを体系的に決める手法。CFDのパラメータスイープやセットアップ探索の効率化に使います。

### 神経演算子（しんけいえんざんし / Neural Operator）
ニューラルネットワークで「関数から関数への写像（演算子）」を学習する手法。FNO（Fourier Neural Operator）やDeepONetが代表例で、CFDや構造解析の「解の場全体」をメッシュに依存せず一度に予測できます。通常のサロゲートモデルより汎用性が高く、異なる解像度のメッシュにも適用できるため、レース車両の空力形状の大量探索に活用が広がっています。

### GNN（グラフニューラルネットワーク / Graph Neural Network）
データを「節点（ノード）と辺（エッジ）」からなるグラフ構造として扱うニューラルネットワーク。FEAメッシュや分子構造のように「点と点がつながった関係」を自然に表現できます。車体形状のSTLデータをそのままグラフに変換してCFD解析結果を予測するサロゲートに利用されており、複雑な局所流れパターンの捕捉でCNNより優れることが示されています。

### デジタルツイン（Digital Twin）
実機・製品の状態をリアルタイムにコンピュータ上で再現した「仮想の双子」。センサーデータと物理シミュレーションを継続的に同期させることで、現在の状態を常に反映します。サスペンション設計変更の仮想検証や走行中の状態監視・故障予知など、学生フォーミュラでも実機テスト回数を減らしながら開発サイクルを短縮するのに使えます。

### 微分可能シミュレータ（びぶんかのうシミュレータ / Differentiable Simulator）
シミュレータ内部の計算を「微分可能」に実装し、機械学習の勾配法でシミュレータを通じた逆伝播ができるようにした手法。モデルのパラメータを実験データから直接・自動的に最適化できるため、タイヤモデル係数や空力モデルの同定精度と速度が大幅に向上します。

### Koopman演算子（こうぷまんえんざんし / Koopman Operator）
非線形なシステムの状態を高次元の特徴空間に持ち上げ、そこでは「線形（1次方程式）」として扱えるようにする数学的な演算子。車両ダイナミクスのような複雑な非線形挙動を線形MPCで近似できるため、計算時間を大幅に削減できます。2026年の論文（arXiv:2606.15094）では走行わずか3周のデータで学習し、MPC計算時間を67%削減しながら追跡精度も28%向上した事例が報告されています。

### MeshGraphNet（メッシュグラフネット）
CFDやFEAのメッシュ（計算格子）データをグラフ構造として扱い、節点間の物理的相互作用をメッセージパッシングで学習するGNNアーキテクチャ。NVIDIAのPhysicsNeMoやXAeroNetの中核技術で、表面圧力・速度場をメッシュ形状に沿って1秒以内に予測します。学生フォーミュラのフロントウィング形状パラメータスタディで、従来CFD1ケース12時間の評価を数百形状でも数分に短縮できます。

### マルチフィデリティ最適化（まるちふぃでりてぃさいてきか / Multi-Fidelity Optimization）
精度の異なる複数のモデル（高精度CFD・簡易サロゲートなど）を組み合わせて設計を最適化する手法。低精度モデルで大量に設計空間を探索し、予測不確かさが高い有望な候補のみ高精度モデルで再評価することで、CFD実行回数を50〜80%削減できます。HPC資源と時間が限られる学生フォーミュラチームの空力・構造設計最適化に特に効果的なアプローチです。

### アクティブラーニング（能動学習 / Active Learning）
機械学習モデルが「次にどのデータを取得すると最も学習効率が上がるか」を自ら選択する学習手法。サロゲートモデルの予測不確かさ（信頼区間）が高い形状のみCFDを追加実行することで、少ないデータで高精度サロゲートを構築できます。限られたHPC割り当てで最大の精度向上を得たい学生チームのサロゲート構築戦略として、マルチフィデリティ最適化と組み合わせて使われます。
