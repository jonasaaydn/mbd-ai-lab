# MBD×AI Lab 集客・普及プレイブック

このサイトを「もっと読まれる」ようにするための実践マニュアル。
コード側の土台（SEO・SNS・計測の仕組み）はすでに実装済み。
ここに書くのは **運営者（あなた）が手を動かす部分** と、その優先順位。

---

## いま読まれていなかった本当の理由

技術記事は十分にある。足りなかったのは「**見つけてもらう導線**」と「**シェアされる仕組み**」。
以下はコード側で対応済み（2026-06-15）：

| 対応済み項目 | 効果 |
|------------|------|
| sitemap.xml 自動生成 | Google が全111記事を見つけられる |
| robots.txt | クローラーへサイトマップの場所を伝える |
| OGP画像（SNSシェア時のカード画像） | X・LINE・Slackで共有した時に見栄えする |
| 構造化データ（JSON-LD） | Google検索でリッチ表示されやすくなる |
| 全ページのcanonical URL | 「全部トップページの複製」と誤認されるバグを修正 |
| フッターのリンク切れ修正 | サイト内の回遊・クロールが正常化 |
| SNSシェアボタン・記事検索 | 読者がシェア／回遊しやすい |
| Google Analytics 差し込み口 | 何が読まれているか計測できる（要・有効化） |

→ **ここから先は「外に出ていく」作業。これがいちばん効く。**

## 優先度1：検索エンジンに登録する（最重要・無料・30分）

検索流入はニッチな技術ブログの生命線。まずここから。

### 1-1. Google Search Console に登録

1. https://search.google.com/search-console を開く
2. 「URL プレフィックス」を選び、`https://jonasaaydn.github.io/mbd-ai-lab/` を入力
3. 所有権の確認方法で「**HTML タグ**」を選ぶ
4. 表示される `<meta name="google-site-verification" content="ここが長い文字列" />` の
   **content の中身だけ** をコピー
5. `src/site.config.ts` の `googleSiteVerification: ''` に貼り付けてコミット＆プッシュ
6. デプロイ完了後（数分）、Search Console の「確認」ボタンを押す
7. 確認できたら、左メニュー「サイトマップ」→
   `https://jonasaaydn.github.io/mbd-ai-lab/sitemap-index.xml` を送信

> 補足：github.io はドメインを他プロジェクトと共有するため、DNS確認は使えません。
> 上の「HTMLタグ方式」が確実です。

### 1-2. Bing Webmaster Tools にも登録（5分）

1. https://www.bing.com/webmasters を開く
2. Search Console から「インポート」すると一発で連携できる
3. Bing経由でChatGPT検索・Copilotにも拾われやすくなる

## 優先度2：コミュニティに記事を届ける（最も即効性がある）

検索で順位が付くまでは数週間〜数ヶ月かかる。その間の読者はコミュニティから来る。
**1記事書いたら、関連コミュニティに1回シェアする** を習慣にする。

### 日本語コミュニティ

| 場所 | 何をするか | 相性 |
|------|----------|------|
| **X (旧Twitter)** | 記事の要点＋数字（「CFDが8時間→数分」）＋リンク | ◎ 最重要 |
| **Qiita / Zenn** | 記事の要約版を投稿し、本文へリンク（クロスポスト） | ◎ 技術者が集まる |
| **note** | 学生フォーミュラ向けの読み物として再構成 | ○ 学生層 |
| **はてなブックマーク** | 記事末尾のB!ボタンから自分でブックマーク | ○ 拡散の起点 |
| **学生フォーミュラのDiscord/Slack** | 該当チームの技術チャンネルで共有 | ◎ ターゲット直撃 |

### 学生フォーミュラ・モータースポーツ系

- **JSAE（自動車技術会）学生フォーミュラ** 関連のSNS・掲示板
- 各大学の自動車部・フォーミュラチームの公式アカウントへ記事を紹介
- **Reddit `r/FSAE`**（英語・世界中の学生フォーミュラ）— 英語要約版を作れば海外流入も狙える

### コツ
- 「ツールの宣伝」ではなく「**あなたの悩みが解決する**」という切り口で書く
- 必ず **Before→After の数字** を添える（クリック率が段違い）
- 記事の `summary` がそのままSNSの説明文になるので、最初の1文を強くする

## 優先度3：アクセスを計測する（改善の前提）

「何が読まれているか」が分からないと改善できない。

### Google Analytics 4 を有効化（無料）

1. https://analytics.google.com でアカウント＋プロパティを作成
2. データストリームに `https://jonasaaydn.github.io/mbd-ai-lab/` を登録
3. 「測定 ID」（`G-` で始まる）をコピー
4. `src/site.config.ts` の `gaMeasurementId: ''` に貼り付けてコミット＆プッシュ
5. 数時間後、リアルタイムにアクセスが表示されれば成功

> プライバシー重視で「Cookie同意バナーを出したくない」なら、
> 代わりに **Cloudflare Web Analytics**（無料・Cookieなし）も選択肢。希望すれば実装します。

### 見るべき指標
- **どの記事が読まれているか** → そのテーマを深掘りした続編を書く
- **流入元**（検索／SNS／直接）→ 効いているチャネルに集中
- **直帰率の高い記事** → 導入文と関連リンクを見直す

## 優先度5：独自ドメイン（任意・年1,000〜2,000円）

`jonasaaydn.github.io/mbd-ai-lab/` よりも `mbd-ai-lab.com` のような独自ドメインの方が：
- 検索順位・信頼性で有利
- 覚えやすく、口コミされやすい
- ルート直下の `robots.txt` が正しく効くようになる

手順の概要（やる場合は手伝います）：
1. お名前.com / Cloudflare 等でドメインを取得
2. GitHub Pages の Settings → Pages → Custom domain に設定
3. DNS の CNAME を設定、`base` を `/` に変更してリンクを調整

## ⚠ 重要：コンテンツ品質の注意（正直な助言）

このサイトは AI が毎日記事を自動生成している。これは強みだが、リスクもある：

- Google は「**大量生成された価値の低いAIコンテンツ**」を検索結果から除外する方針
  （スケーリングされたコンテンツの不正利用ポリシー / Helpful Content）
- 量だけ増やすと、ドメイン全体の評価が下がる可能性がある

**対策：このサイトの本当の武器は「学生フォーミュラへの具体的応用」という独自性。**
- 実際に動くコード・実測の数字・チーム視点 — これは他にないので評価されやすい
- 「ニュースの寄せ集め」より「**この悩みをこう解決した**」記事を厚くする
- 自動生成記事も、たまに人の目で手を入れる（体験談・写真・実データ）と信頼性が上がる
- 1日6本の量産より、**1日2本でも応用例が濃い**方が長期的に有利
